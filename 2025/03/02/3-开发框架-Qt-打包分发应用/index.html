静态编译 vs 动态编译

核心差异对比

特性 静态编译 动态编译
可执行文件 单一大文件（50-100MB+） 小文件 + 多 DLL（总计相近）
插件支持 ❌ 不支持 ✅ 完整支持
更新灵活性 需重新编译整个应用 可独立更新 Qt 库
内存占用 每进程独立加载，多进程时较高 共享库，多进程时更高效
部署复杂度 简单（单文件） 需管理 DLL 依赖
LGPL 合规 需提供对象文件或源码 直接使用，无额外义务

静态编译 Qt (从源码构建)

静态编译需从 Qt 源码重新构建，过程耗时较长（2-4 小时），建议仅在必要时使用。

Windows: windeployqt 详解

windeployqt 是 Qt 官方提供的 Windows 部署工具，自动收集所有运行时依赖。

基础用法

完整参数参考

参数 说明
--release / --debug 指定构建类型
--qmldir <path> QML 应用必须指定，扫描 QML 导入
--compiler-runtime 复制 MSVC/MinGW 运行时库
--no-translations 不复制翻译文件（减小体积）
--no-opengl-sw 不复制软件 OpenGL 库
--no-system-d3d-compiler 不复制 D3D 编译器
--list <option> 列出依赖 (source/target/relative/mapping)
--dir <path> 指定输出目录
--verbose <level> 详细输出 (0-2)

特性 Qt IFW Inno Setup

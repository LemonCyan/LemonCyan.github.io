Qt应用打包与分发

把Qt应用交到用户手里，要过几道关：编译方式怎么选，依赖怎么收集，平台怎么配置，安装包怎么做。本文按这个顺序说，Windows、macOS、Linux都涉及。

编译方式选择

先决定用静态编译还是动态编译。两者在部署难度、文件大小、许可证要求上差别不小。

静态编译与动态编译对比

特性 静态编译 动态编译
可执行文件 单一大文件（50-100MB+） 小文件 + 多DLL（总计差不多）
插件支持 不支持 支持
更新方式 重新编译整个应用 可单独更新Qt库
内存占用 每个进程独立加载，多进程时占用高 共享库，多进程时省内存
部署难度 简单，就一个文件 要管理DLL依赖
LGPL合规 必须提供目标文件或源码 直接用，没额外要求

怎么选？

  • 内部工具、嵌入式设备 → 静态编译，省事
  • 商业软件或者要用插件（数据库驱动、图片解码器）→ 动态编译
  • 开源项目 → 都行，动态编译更符合LGPL的本意

Qt给每个平台都准备了部署工具，自动把运行时依赖收拢到一起。

Windows: windeployqt

windeployqt是Qt官方出品，自动找齐所有dll。

基本用法

常用参数

参数 作用
--release / --debug 指定编译类型
--qmldir <path> QML应用必须加，扫描QML导入
--compiler-runtime 复制MSVC/MinGW运行库
--no-translations 不复制翻译文件，包小点
--no-opengl-sw 不复制软件OpenGL
--no-system-d3d-compiler 不复制D3D编译器
--list <option> 列依赖（source/target/relative/mapping）
--dir <path> 输出目录
--verbose <level> 详细程度（0-2）

IFW和Inno Setup怎么选

特性 Qt IFW Inno Setup
跨平台 Windows/macOS/Linux 只有Windows
在线更新 自带 要另找方案
组件化 支持 基本不行

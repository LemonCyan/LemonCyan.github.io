Qt MySQL 全流程实战指南

1. 核心机制与架构 (Architecture)

与 SQLite 的嵌入式架构不同，Qt 操作 MySQL 遵循标准的客户端-服务器 (C/S) 模型。理解这一物理隔离对于解决”驱动加载失败”问题至关重要。

Qt MySQL 驱动栈

  1. Qt Application: 你的应用程序。
  2. QMYSQL Plugin: Qt 提供的桥接插件 (qsqlmysql.dll)。注意：这只是一个包装器，它本身不包含 MySQL 协议实现。
  3. MySQL Connector (libmysql): 真正的 Oracle 官方客户端库。这是最常缺失的一环。

[!IMPORTANT]
99% 的 “QMYSQL driver not loaded” 错误，是因为 Qt 插件找到了，但插件所依赖的 libmysql.dll 或 libssl 找不到。

2.2 终极解决方案 (Windows)

现象：available drivers 列表中没有 QMYSQL，或者连接时提示 Driver not loaded。

修复步骤：

  1. 定位位数：确认你的 Qt 编译器是 64位 (MSVC2019_64) 还是 32位 (MinGW_32)。必须下载对应位数的 MySQL Connector。
    • 误区：安装了 64位 MySQL Server，但 Qt 程序是 32位的 -> 驱动加载失败。
  2. 下载依赖：
    • 访问 MySQL Connector/C Downloads。
    • 下载对应的 Zip 包（无需安装）。
  3. 复制文件：
    • 解压后，将 lib 目录下的 libmysql.dll 复制到你的 Qt 编译输出目录 (即 .exe 所在目录)。
    • 注意：如果需要 SSL 连接，还需要 OpenSSL 库 (libssl-1_1-x64.dll, libcrypto-1_1-x64.dll)，通常位于 Qt 的 bin 目录中，确保它们也在搜索路径内。

3. 工程化连接配置 (Production Connection)

3.1 稳健连接模式

MySQL 连接可能会因为网络波动或服务器超时（wait_timeout）而断开。Qt 提供了断线重连选项。

4. 高效 CRUD 与安全性 (Best Practices)

4.1 绝对禁止的操作

[!WARNING]
严禁使用 QString::arg() 拼接 SQL。

  • ❌ 注入漏洞：User: "admin' --"
  • ❌ 性能低下：每次执行都需要服务器重新解析 SQL 语法树。

4.2 事务与预处理 (Transactions & Prepared Statements)

MySQL 的网络延迟远高于 SQLite 的本地磁盘 I/O。批量操作必须使用事务。

4.3 存储过程调用

MySQL 强大的地方在于存储过程。Qt 可以通过 call 语法调用。

5.3 线程安全

与 SQLite 一样，**QSqlDatabase 连接不能跨线程共享。
如果是多线程高并发场景（如 HTTP 服务器），建议实现一个连接池 (Connection Pool)**，维护一组打开的连接供工作线程复用，而不是频繁地 Open/Close。

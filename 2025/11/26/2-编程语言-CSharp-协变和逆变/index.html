协变和逆变

C# 协变与逆变：泛型类型兼容的核心机制（面向 C++ 程序员）

C# 中的协变（Covariance） 和逆变（Contravariance） 是针对 泛型接口 / 委托 的特性，核心目的是 在类型安全的前提下，支持泛型类型的隐式转换—— 简单说，就是让 IFoo<子类> 和 IFoo<父类> 之间能合理兼容，解决默认泛型 “不变性” 导致的灵活性不足问题。

对 C++ 程序员来说，这是 C# 泛型独有的特性（C++ 模板无直接对应，但可类比 C++ 函数指针的类型兼容规则、C++20 concepts 的约束逻辑），其设计核心是 “类型安全优先”：所有协变 / 逆变转换都必须经过编译器验证，确保不会出现类型转换异常（如将 Dog 存入 List<Cat>）。

一、先搞懂：默认泛型是 “不变的”（为什么需要协变 / 逆变？）

在理解协变 / 逆变前，必须先明确：C# 泛型默认是 不变的（Invariant）—— 即 List<Cat>（子类泛型）和 List<Animal>（父类泛型）之间没有任何隐式转换关系，即使 Cat 是 Animal 的子类。

正是为了避免这种 “类型污染”，C# 默认禁止泛型类型的直接转换 —— 但这也带来了灵活性问题：有些场景下，List<Cat> 确实 “可以当作”List<Animal> 使用（比如只读取元素，不修改），此时默认不变就显得过于严格。

协变和逆变的核心作用：在 “不破坏类型安全” 的特定场景下，允许泛型类型的隐式转换。

二、协变（Covariance）：子类泛型 → 父类泛型（out 关键字）

1. 核心定义

协变是指：泛型接口 / 委托支持从 “子类泛型” 隐式转换为 “父类泛型”（转换方向与类继承方向一致：Cat → Animal → IFoo<Cat> → IFoo<Animal>）。

  • 标记：接口用 out 关键字修饰泛型参数（如 IEnumerable<out T>）；
  • 核心约束：泛型参数 T 只能作为 方法的返回值 / 只读属性（不能作为方法参数）—— 确保 “只出不进”，避免类型污染。

3. 协变的类型安全保障：out 关键字的意义

out 关键字强制约束：泛型参数 T 只能 “向外输出”（返回值 / 只读属性），不能 “向内输入”（方法参数）。这样一来，协变转换后无法修改原集合（如添加 Dog），自然不会出现类型污染。

三、逆变（Contravariance）：父类泛型 → 子类泛型（in 关键字）

1. 核心定义

逆变是指：泛型接口 / 委托支持从 “父类泛型” 隐式转换为 “子类泛型”（转换方向与类继承方向相反：Animal → Cat → IFoo<Animal> → IFoo<Cat>）。

  • 标记：接口用 in 关键字修饰泛型参数（如 IComparer<in T>）；
  • 核心约束：泛型参数 T 只能作为 方法的输入参数（不能作为返回值 / 只读属性）—— 确保 “只进不出”，避免类型不匹配。

四、协变与逆变的核心规则（避坑关键）

1. 标记规则：out = 协变，in = 逆变

特性 关键字 转换方向 泛型参数使用约束 典型例子
协变 out 子类泛型 → 父类泛型 仅作为返回值 / 只读属性 IEnumerable、Func
逆变 in 父类泛型 → 子类泛型 仅作为方法参数 IComparer、Action
不变 无 无隐式转换 可作为参数 / 返回值 List、Dictionary<TKey,TValue>

2. 适用范围：仅支持接口和委托

协变 / 逆变 只适用于泛型接口和泛型委托，泛型类（如 List<T>、Dictionary<T>）和泛型结构体不支持显式协变 / 逆变（默认不变）。

比如 List<T> 不能标记为协变，因为它的 Add(T) 方法允许修改集合（T 作为参数），this[int index] 既可以读（返回 T）也可以写（接收 T），不符合 out 或 in 的约束。

3. 类型限制：仅支持引用类型

协变 / 逆变的转换只支持引用类型，值类型（如 int、DateTime）不支持。例如：

原因：值类型在泛型中会被 “具体化”（如 List<int> 和 List<object> 是完全独立的类型，无继承关系），而引用类型的泛型实例共享相同的接口结构。

4. 类型安全：编译器强制校验

所有协变 / 逆变转换都由编译器验证，确保不会出现运行时类型异常。核心逻辑是：

  • 协变（out）：只能 “读”（返回值），不能 “写”（参数）→ 不会污染原集合；
  • 逆变（in）：只能 “写”（参数），不能 “读”（返回值）→ 父类逻辑适用于子类。

3. 核心差异

  • C#：通过 out/in 关键字显式标记协变 / 逆变，仅适用于泛型接口 / 委托，编译器严格校验；
  • C++：无显式标记，通过函数指针 / 成员函数指针的类型兼容规则隐式支持，适用于函数类型，无泛型层面的直接对应。

总结

协变和逆变的核心是 “在类型安全的前提下，提升泛型接口 / 委托的灵活性”：

  1. 协变（out）：子类泛型 → 父类泛型，仅支持返回值，适用于 “只读” 场景（如遍历）；
  2. 逆变（in）：父类泛型 → 子类泛型，仅支持参数，适用于 “通用处理” 场景（复用父类逻辑）；
  3. 不变：默认行为，泛型类 / 结构体均为不变，支持读写操作，但无隐式转换；
  4. 与 C++ 类比：类似函数指针的类型兼容规则，但 C# 是泛型层面的显式特性，更安全、更易复用。

对 C++ 程序员来说，无需纠结底层实现，记住核心：out 是 “输出”（协变），in 是 “输入”（逆变），一切转换都以类型安全为前提—— 编译器会帮你校验，只要不违反 out/in 的约束，就能放心使用。

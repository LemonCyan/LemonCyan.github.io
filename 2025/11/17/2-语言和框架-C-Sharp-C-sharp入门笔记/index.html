.NET 框架

C# 是一门编程语言，而 .NET 是一个开发平台和运行时环境，.Net 框架由一个巨大的代码库组成。包含：

运行时（CLR：Common Language Runtime）

  • 管理内存（垃圾回收）
  • 管理线程
  • 执行 IL 中间语言
  • 提供类库
    基类库（BCL）
  • 文件读写（System.IO）
  • 网络通信（System.Net）
  • 容器（List、Dictionary）
  • 反射（Reflection）
    开发框架
  • ASP.NET（做 Web）
  • WinForms、WPF、MAUI（做桌面 GUI）
  • Entity Framework（做数据库）

程序结构

  • 命令空间 namespace （用于分类代码并避免类名、接口名冲突，可以嵌套）
    • 使用 using 关键字
    • 使用 . 全饰名称
  • 函数入口 Main（顶级语句）
  • Class 方法
  • Class 属性/数据成员
  • 变量作用域
    • 局部变量
    • 全局变量
    • static 变量

数据类型

  • 值类型

  • 引用类型
    指向内存位置，不包含存储在变量中的实际数据，包含对变量的引用

    • object 类型
    • dynamic 类型（在运行时推断变量的类型）
    • string 类型

  • 指针类型

  • 常量（const）

  • class 类型（typeof (类变量)返回 class 类型）

  • 可空类型（值/引用）

    • ? 让值类型可以为 null。 int? x; Nullable<int> x = null;// 默认值为 null
    • ?? 当左侧的值为 null 时，?? 返回右侧的默认值 int result = i ?? 0;

类型转换

  • 隐式类型转换
    编译器自动进行，从小的数据范围到大的数据范围，不会导致数据丢失，例如从小的整数类型转换为大的整数类型，从派生类转换为基类
  • 显式类型转换
    需要进行类型转换的兼容性检查和类型转换的安全性检查
  • 操作符 is 判断
  • 操作符 as 安全类型转换（转换失败为 null）
  • 自定义类型转换
    • 关键字 implicit 隐式转换
    • 关键字 explicit 显式转换

  • 赋值 array[0] = 1;

  • 多维数组 int[,,] array;//三维数组

  • 交错数组 int[][] array = new int[5][]; array[0] = new int[2];

  • 传递数组给函数

  • 参数数组 params 关键字

  • Array 类

  • ?[] 用于访问索引器或数组元素 array?[10]
  • ?: 条件判断 Exp1 ? Exp2 : Exp3;

  • for/foreach

  • continue

方法

  • 递归调用
  • 参数传递
    • 值传递
    • 引用传递（ref）
    • 输出传递（out）
  • 扩展方法 ，必须是静态的，必须在静态类中定义

封装

  • public：所有对象都可以访问；
  • private：对象本身在对象内部可以访问；
  • protected：只有该类对象及其子类对象可以访问
  • internal：同一个程序集（exe 或 dll）的对象可以访问；
  • protected internal：访问限于当前程序集或派生自包含类的类型。

继承

  • 一个类可以继承自另一个类，被称为基类（父类）和派生类（子类）
  • 一个类可以继承多个接口，但只能继承自一个类。
  • 关键字 base，调用基类的构造函数和方法

多态

  • 多态就是同一个接口，使用不同的实例而执行不同操作
  • overload 在静态多态性中，函数的响应是在编译时发生的。
    • 函数重载，同一个范围内对相同的函数名有多个定义（参数类型或个数不同，与返回类型无关）
    • 操作符重载，public static Box operator+ (Box b, Box c)
  • override 在动态多态性中，函数的响应是在运行时发生的。
    • abstract 抽象类 abstract class Shape
    • virtual 当有一个定义在类中的函数需要在继承类中实现时，可以使用虚方法

类和结构

结构体

  • 结构可带有方法、字段、索引、属性、运算符方法和事件，用于表示轻量级数据。
  • 结构可定义构造函数，但不能定义析构函数。但不能为结构定义无参构造函数。无参构造函数(默认)是自动定义的，且不能被改变。
  • 结构不能继承其他的结构或类，也不能作为其他结构或类的基础结构（不能被继承）。
  • 结构可以继承接口，可实现一个或多个接口。
  • 结构成员不能指定为 abstract、virtual 或 protected。
  • 当使用 New 操作符创建一个结构对象时，会调用适当的构造函数来创建结构。与类不同，结构可以不使用 New 操作符即可被实例化。如果不使用 New 操作符，只有在所有的字段都被初始化之后（显示赋值，防止未初始化的字段使用），字段才被赋值，对象才被使用。
  • 结构变量通常分配在栈上，这使得它们的创建和销毁速度更快。但是，如果将结构用作类的字段，且这个类是引用类型，那么结构将存储在堆上。
  • 结构默认情况下是可变的，这意味着你可以修改它们的字段。但是，如果结构定义为只读，那么它的字段将是不可变的。

类

  • 构造函数与类的名称完全相同，它没有任何返回类型。
  • 析构函数在类的名称前加上 ~，无返回值，参数。析构函数用于在结束程序（比如关闭文件、释放内存等）之前释放资源。析构函数不能继承或重载
  • 静态成员，声明一个类成员为静态时，意味着无论有多少个类的对象被创建，只会有一个该静态成员的副本，可以通过直接调用类而不需要创建类的实例来获取

区别

值类型 vs 引用类型：

  • 结构是值类型（Value Type）： 在栈上分配内存。传递给方法或赋值时，执行深拷贝。
  • 类是引用类型（Reference Type）： 在堆上分配内存。传递给方法或赋值时，执行浅拷贝

继承和多态性：

  • 结构不能继承： 结构不能继承其他结构或类，也不能作为其他结构或类的基类。
  • 类支持继承： 类支持继承和多态性，可以通过派生新类来扩展现有类的功能。

默认构造函数：

  • 结构不能有无参数的构造函数
  • 类可以有无参数的构造函数

赋值行为：

  • 类型为类的变量在赋值时存储的是引用，因此两个变量指向同一个对象。
  • 结构变量在赋值时会复制整个结构，因此每个变量都有自己的独立副本。

传递方式：

  • 类对象在方法调用时通过引用传递，在方法中对对象所做的更改会影响到原始对象。
  • 结构对象通过值传递，传递的是结构的副本。在方法中对结构所做的更改不影响原始对象

可空性：

  • 结构体是值类型，不能直接设置为 null： 可以使用 Nullable<T> 或称为 T? 的可空类型
  • 类是引用类型，可为 null

性能和内存分配：

  • 结构通常更轻量： 值类型且在栈上分配内存，比类更轻量，适用于简单的数据表示。
  • 类可能有更多开销： 由于类是引用类型，可能涉及更多的内存开销和管理。

接口

  • 接口定义 interface IMyInterface
  • 如果一个接口继承其他接口，那么实现类或结构就需要实现所有接口的成员

异常处理

  • try：一个 try 块标识了一个将被激活的特定的异常的代码块。后跟一个或多个 catch 块。
  • catch：程序通过异常处理程序捕获异常。catch 关键字表示异常的捕获。
  • finally：finally 块用于执行给定的语句，不管异常是否被抛出都会执行。例如，如果您打开一个文件，不管是否出现异常文件都要被关闭。
  • throw：当问题出现时，程序抛出一个异常。使用 throw 关键字来完成。
  • 用户自定义的异常类是派生自 ApplicationException 类，eg

文件 IO

  • System.IO 命名空间有各种不同的类，用于执行各种文件操作，如创建和删除文件、读取或写入文件，关闭文件等
  • 文本文件 StreamReader 和 StreamWriter
  • 二进制文件 BinaryReader 和 BinaryWriter
  • 文件和目录 FileInfo 和 DirectoryInfo

特性

  • 用于添加元数据，附加到代码上的描述信息，是一个类，继承自 System.Attribute
  • 可以附加到类、方法、属性、字段等
  • 语法 [attribute(positional_parameters, name_parameter = value, ...)]element

  • 元数据，程序本身也是数据，有关程序及其类型的数据被称为元数据

    • 审查集合中的各种类型，以及实例化这些类型。
    • 延迟绑定的方法和属性（property）。
    • 在运行时创建新类型，然后使用这些类型执行一些任务。

属性

  • 类和结构体中用于封装数据的成员，看作是对字段的包装，隐藏字段的内部实现细节，通常由 get 和 set 访问器组成
  • 只读属性，省略 set 访问器
  • 只写属性，省略 get 访问器
  • get 和 set 中可以包含自定义逻辑，eg. 可以增加数据非法校验 throw new ArgumentException("Name cannot be empty.");
  • 抽象属性，抽象类可拥有抽象属性，这些属性应在派生类中被实现

索引器

  • 允许一个对象可以像数组一样使用下标的方式进行访问
  • 索引器定义的时候不带有名称，但带有 this 关键字，它指向对象实例
  • 将一个类定义为一个索引器时，该类的行为像一个数组一样，可以通过下标访问类的成员
  • 索引器的生命类似于属性，可以使用 get 和 set 来定义索引器，属性返回/设置一个特定成员数据，索引器返回/设置对象实例的一个特定值

重载索引器

索引器（Indexer）可被重载。索引器声明的时候也可带有多个参数，且每个参数可以是不同的类型。没有必要让索引器必须是整型的。C# 允许索引器可以是其他类型，例如，字符串类型。

委托

  • 类型安全的函数指针，只有匹配的方法才可以赋值给委托，赋值后允许将方法作为参数进行传递，帮助实现事件驱动的编程、回调机制和函数式编程风格
  • 存有对某个方法的引用的引用类型变量，可在运行时被改变
  • 用于事件处理，回调函数，LINQ 等操作
  • 所有的委托都继承自 System.Delegate 类
  • 委托对象需要通过 new 关键字进行创建
  • 委托对象可以通过 + 进行合并，通过 - 移除委托，只用同类型的委托可以合并
  • 委托的多播，一个委托被调用时要调用的方法的调用列表，多播委托（Multicast Delegate）调用顺序是严格按照添加顺序执行的
  • 委托常常和事件一起使用，事件是一种特殊类型的委托
  • 事件本质上是一个封装了委托的类型，用于响应程序中的某些操作
  • 异步调用：可以将委托与 BeginInvoke 和 EndInvoke 方法一起使用，进行异步调用
  • 可以用匿名方法和 lambda 表达式创建委托

委托作为参数传递

委托类型 系统内置委托可以省去声明，简化语法

  • 自定义委托
  • Action：代表不返回值的方法。可以接受最多 16 个参数。
  • Func：代表有返回值的方法。最多接受 16 个参数，第一个参数是输入参数，最后一个参数是返回值类型。
  • Predicate：代表返回 bool 值的方法，通常用于条件判断。

事件

  1. 声明委托 定义事件将使用的委托类型，委托是一个函数签名
  2. 声明事件 使用 event 关键字声明一个事件
  3. 触发事件 在适当的时候调用事件，通知所有订阅者
  4. 订阅和取消订阅事件 其他类可以通过 += 和 -= 订阅和取消订阅事件
  • event 是一种成员，用于将特定的事件通知发送给订阅者，允许一个对象将状态的变化通知其他对象，而不需要知道这些对象的实现细节，C#中使用事件机制实现线程间的通讯
  • 事件在类中声明且生成，通过使用同一个类或其他类中的委托与事件处理程序相关联，包含事件的类用于发布事件，称为发布器。其他接受该事件的类称为订阅器
  • 发布器是一个包含事件和委托定义的对象，事件和委托之间的联系也定义在这个对象中。发布器类的对象调用这个事件，并通知其他对象
  • 订阅器是一个接受事件并提供事件处理程序的对象，在发布器类中的委托调用订阅器类中的方法（事件处理程序）

集合

  • 专门用于数据存储和检索的类

  • 泛型集合是类型安全、高性能的首选集合，核心优势是编译时类型检查（避免类型转换错误）、无需装箱拆箱（提升性能）、明确元素类型（增强代码可读性）

  • List 列表 new List<int>();

  • Dictionary 字典 new Dictionary<TKey,TValue>();

  • ArrayList 动态数组，大小可变，通过索引访问

  • Hashtable 哈希表，通过键值对访问

  • SortedList 排序列表，使用索引和键值对进行访问，是数组和哈希的组合

  • Stack 栈，后进先出

  • Queue 队列，先进先出

  • BitArray 二进制数组 BitArray bits = new BitArray(8, false);

  • 随机访问频繁 → 选 List<T> 或 Dictionary<TKey,TValue>；

  • 频繁插入 / 删除（非首尾）→ 选 LinkedList<T>；

  • 去重或集合运算 → 选 HashSet<T> 或 SortedSet<T>；

  • 多线程场景 → 选 System.Collections.Concurrent 下的集合。

泛型

  • 允许延迟编写类或方法中的数据类型
  • 有助于代码复用，类型安全和提高性能
  • 创建泛型集合
  • 创建泛型接口，泛型类，泛型方法，泛型事件，泛型委托
  • 对泛型类进行约束以访问特定数据类型的方法
  • 泛型数据类型中的类型信息可以在运行中用反射获取

匿名方法

不安全代码

指的是使用了指针变量的代码块。由于 C#中声明的变量在内存中的存储受垃圾回收器管理；因此一个变量（例如一个大数组）有可能在运行过程中被移动到内存中的其他位置。如果一个变量的内存地址会变化，那么指针也就没有意义了。解决方法就是使用 fixed 关键字来固定变量位置不移动。

stackalloc 在 unsafe 不安全环境中，我们可以通过 stackalloc 在堆栈上分配内存，因为在堆栈上分配的内存不受内存管理器管理，因此其相应的指针不需要固定。

多线程

  • 线程生命周期开始于 System.Threading.Thread 类的对象被创建时
  • 程序开始执行时，主线程自动创建。使用 Thread 类创建的线程被主线程的子线程调用。您可以使用 Thread 类的 CurrentThread 属性访问线程。
  • 线程是通过扩展 Thread 类创建的。扩展的 Thread 类调用 Start() 方法来开始子线程的执行
  • Thread 类提供了各种管理线程的方法，sleep
  • Abort() 方法用于销毁线程。通过抛出 threadabortexception 在运行时中止线程。这个异常不能被捕获，如果有 finally 块，控制会被送至 finally 块。

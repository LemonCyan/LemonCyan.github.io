Qt 定时器完全指南：QTimer 从入门到精通

核心概念

Qt 的定时器系统基于事件循环机制运作。QObject 作为所有 Qt 对象的基类，提供了基础的定时器支持。定时器通过周期性触发事件来执行预定操作，其工作依赖于应用程序的事件循环正常运行。

事件循环依赖

定时器事件只有在事件循环运行时才能传递。必须调用 QCoreApplication::exec() 启动事件循环后，定时器才能正常工作。当应用程序忙于处理其他事件时，定时器触发可能会延迟——定时器的准确性取决于应用程序的响应粒度。

定时器生命周期

使用 QObject::startTimer() 启动定时器时：

  • 以毫秒为单位指定触发间隔
  • 返回唯一的整数定时器 ID
  • 定时器持续触发直到显式调用 QObject::killTimer() 终止

QTimer 基础用法

QTimer 是 Qt 提供的高级定时器 API，继承自 QObject，通过信号槽机制提供简洁的编程接口。

多线程环境

线程亲和性规则

Qt 使用线程亲和性（Thread Affinity） 确定由哪个线程发出 timeout() 信号。核心规则：

操作 线程要求
创建定时器 任意线程
启动/停止定时器 必须在定时器所属线程
接收 timeout 信号 由定时器所属线程发出

[!CAUTION]
不能从其他线程启动或停止定时器。违反此规则将导致未定义行为。

定时器精度与类型

定时器精度受底层操作系统和硬件影响，可通过 Qt::TimerType 枚举控制精度级别。

精度类型对比

类型 精度目标 适用场景
Qt::PreciseTimer 毫秒级精度 动画、实时处理
Qt::CoarseTimer ±5% 误差范围 常规业务逻辑
Qt::VeryCoarseTimer 秒级精度（四舍五入） 超时检测、心跳

平台特性

UNIX 系统（Linux、macOS、iOS）：

  • PreciseTimer 保持毫秒精度
  • CoarseTimer 会自动对齐相近触发时间的定时器，减少 CPU 唤醒次数以降低功耗

Windows 系统：

  • PreciseTimer 使用多媒体定时器（Multimedia Timer）
  • CoarseTimer 和 VeryCoarseTimer 使用标准 Windows 定时器

精度限制

平台 典型精度
Windows 2000 15 毫秒
现代 Linux/macOS 1 毫秒

间隔上限： 由有符号整数的毫秒数决定，约为 24.8 天。

[!IMPORTANT]
系统繁忙时，所有类型定时器都可能延迟触发。Qt 保证即使错过多次超时也只发出一次 timeout 信号，随后恢复正常间隔。

小结

要点 说明
事件循环 定时器依赖事件循环，无循环则不工作
线程安全 必须在定时器所属线程启停
精度控制 三级精度类型适应不同场景
替代方案 QBasicTimer 轻量但功能受限
资源管理 利用对象树机制自动释放

[!NOTE]
部分操作系统对定时器数量有限制，Qt 内部会尝试规避这些限制。在高频定时器场景下，建议测试目标平台的实际表现。

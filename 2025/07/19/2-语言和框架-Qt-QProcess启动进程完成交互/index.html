QProcess启动进程完成交互

运行进程

要启动进程，请将要运行的程序的名称和命令行参数作为参数传递给 start ()。参数在 QStringList 中以单独的字符串形式提供。

然后，QProcess 进入 Start 状态，当程序启动后，QProcess 进入 Running 状态并发出 start ()。

QProcess 允许您将进程视为顺序 I/O 设备。您可以对进程进行写入和读取，就像使用 QTcpSocket 访问网络连接一样。然后，您可以通过调用 write () 对进程的标准输入进行写入，并通过调用 read ()、readLine () 和 getChar () 读取标准输出。由于 QProcess 继承自 QIODevice, 因此也可以用作 QXmlReader 的输入源，或者用于生成要使用 QNetworkAccessManager 上传的数据。

当进程退出时，QProcess 重新进入 NotRunning 状态 (初始状态), 并发出 finited ()。

Finished () 信号提供进程的退出代码和退出状态作为参数，还可以调用 exitCode () 获取最后一个完成的进程的退出代码，以及 exitStatus () 获取其退出状态。如果在任何时间点发生错误，QProcess 将发出 errorOccurred () 信号。还可以调用 error () 查找最后发生的错误类型，以及 state () 查找当前进程状态。

注意：VxWorks、iOS、Apple tvOS、watchOS 或通用 Windows 平台不支持 QProcess。

通过信道进行通信

进程有两个预定义的输出通道：标准输出通道 (stdout) 提供常规控制台输出，标准错误通道 (stderr) 通常提供进程打印的错误。这些通道代表两个独立的数据流。可以通过调用 setReadChannel () 在它们之间切换。当当前读取通道上有数据可用时，QProcess 会发出 readyRead ()。当有新的标准输出数据可用时，它也会发出 readyReadStandardOutput (); 当有新的标准错误数据可用时，它会发出 readyReadStandardError ()。除了调用 read ()、readLine () 或 getChar () 之外，还可以通过调用 readAllStandardOutput () 或 readAllStandardError () 显式读取两个通道中任何一个的所有数据。

通道的术语可能会产生误导。请注意，进程的输出通道对应于 QProcess 的读通道，而进程的输入通道对应于 QProcess 的写通道。这是因为我们使用 QProcess 读取的是进程的输出，而我们写入的则成为进程的输入。

QProcess 可以合并两个输出通道，使得来自运行进程的标准输出和标准错误数据都使用标准输出通道。在启动进程之前，使用 MergedChannel 调用 setProcessChannelMode () 来激活这个功能。你还可以选择将运行进程的输出转发给调用进程，即主进程，方法是将 ForwardedChannel 作为参数传递。也可以只转发其中一个输出通道 —— 通常一个通道会使用 ForwardedErrorChannel, 但 ForwardedOutputChannel 也存在。请注意，在 GUI 应用程序中使用通道转发通常是个坏主意 —— 你应该以图形方式呈现错误。

某些进程需要特殊的环境设置才能运行。可以通过调用 setProcessEnvironment () 来设置进程的环境变量。要设置工作目录，请调用 setWorkingDirectory ()。默认情况下，进程在调用进程的当前工作目录中运行。

属于使用 QProcess 启动的 GUI 应用程序的视窗定位和屏幕 Z 顺序由底层视窗系统控制。对于 Qt 5 应用程序，可以使用 - qwindowgeometry 命令行选项指定定位；X11 应用程序通常接受 - geometry 命令行选项。

注意：在 QNX 上，由于操作系统的限制，设置工作目录可能会导致除 QProcess 调用线程外的所有应用程序线程在生成过程中暂时冻结。

同步进程 API

QProcess 提供了一组函数，允许在没有事件循环的情况下使用它，这些函数通过挂起调用线程直到发出特定信号来实现：

  • WaitForStarted () 会阻塞，直到进程启动。
  • WaitForReadyRead () 会阻塞，直到当前读取通道上有新数据可用于读取。
  • WaitForBytesWriten () 会阻塞，直到向进程写入一个数据负载。
  • WaitForFinished () 会阻塞，直到进程完成。

从主线程 (调用 QApplication:：exec () 的线程) 调用这些函数可能会导致用户界面冻结。

Windows 用户注意事项
一些 Windows 命令 (例如 dir) 不是由单独的应用程序提供的，而是由命令解释器本身提供的。如果尝试使用 QProcess 直接执行这些命令，它将不起作用。一种可能的解决方案是自行执行命令解释器 (在一些 Windows 系统上为 cmd.exe), 并要求解释器执行所需的命令。
另请参阅 QBuffer、QFile 和 QTcpSocket。

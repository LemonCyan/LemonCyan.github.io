Qt容器

容器分类

Qt 中的容器和标准的 STL 容器相似，但在使用过程中有显著的区别。Qt 提供的容器更轻量、更安全且易于使用，具备隐式数据共享、不可变特性，经过优化以提高速度并降低内存占用行为。最重要的是，这些容器是线程安全的。这些容器类是平台无关的，不会因编译器不同而有不同的实现。

隐式数据共享被称为”写时复制”（copy on write）机制，当在传递容器作为参数时，不会产生额外的性能损失。

容器类型

以下是主要的容器类型及其实现特性：

QList<T>

  • 基于数组实现，提供快速的向前添加和向后追加操作。
  • 适用于需要频繁读取和末尾追加元素的场景，然而在中间插入数据时性能较低。

QLinkedList<T>

  • 基于链表实现，适用于频繁插入或删除元素的场景。

QVector<T>

  • 在相邻的内存中存储给定数据类型 T 的一组数值，类似于 QList，但通常在类中使用，如 QStack 和 QQueue。
  • 在一个 QVector 的前部或者中间位置进行插入操作的速度是很慢的，这是因为这样的操作将导致内存中的大量数据被移动，这是由 QVector 存储数据的方式决定的。

QMap<key, T>

  • 基于二叉查找树实现，支持对键有序存储的快速查找，QMap 总是按照键 Key 的顺序存储数据。 通过 insertMulti 或 QMultiMap 提供存储一键多值的情况
  • QMap 的键类型 Key 必须提供 operator<() 函数

QHash<key, T>

  • 基于哈希表实现，QHash 具有比 QMap,更快的查找速度。QHash 以任意的顺序存储数据项。当存储的顺序无关紧要时，建议使用 QHash 作为存放数据的容器。QMultiHash 也可以存储一键多值的情况。
  • QHash 的键类型 Key 必须提供 operator==() 和一个全局的 qHash(Key)函数

QStack<T>

  • LIFO（后进先出）结构，常用于堆栈操作。

Queue<T>

  • FIFO（先进先出）结构，常用于队列操作。

容器迭代器

Qt 提供了多种迭代器类型，分别适用于不同的场景和需求。了解它们的特点、优势和使用场景对于高效编程至关重要。

Qt 的容器类为遍历其中的内容提供了以下两种方法。

  • Java 风格的迭代器(Java-style iterators)。
  • STL 风格的迭代器(STL-styleiterators)，能够同 Qt 和 STL 的通用算法一起使用，并且在效率上也略胜一筹。

Java 风格迭代器

Java 风格迭代器是 Qt4 中引入的一组迭代器，它们以简洁易用的 API 著称，类似于 Java 的迭代器。这些迭代器通过指向容器中两个元素之间的”位置”来实现遍历，可能位于第一个元素之前（类似于 STL 的 begin()）或最后一个元素之后（类似于 STL 的 end()）。虽然这种风格的迭代器易于使用，但相比 STL 风格迭代器，它们的效率较低，因为它们通常需要更多的间接访问。

以下是常见的 Java 风格迭代器：

  • QListIterator: 适用于 QList 的迭代器。
  • QMapIterator: 适用于 QMap 的迭代器。
  • QDictionaryIterator: 适用于 QDictionary 的迭代器（已废弃，建议使用 QMap）。

示例：遍历 QList

STL 风格迭代器

STL 风格迭代器从 Qt 2.0 开始支持，设计目标是与 STL（标准模板库）迭代器接口兼容。这些迭代器直接指向容器中的元素，支持更高效的随机访问和修改。他们的性能和语法与 STL 容器的迭代器高度一致，因此 对熟悉 C++ STL 的开发者非常友好。

每种容器都提供了以下几种迭代器：

  • 输入迭代器：只读访问。
  • 输出迭代器：只写访问。
  • 前向迭代器：支持读写并可以多次遍历。
  • 双向迭代器：支持前后移动和读写。
  • 随机访问迭代器：支持直接跳转到任意位置。

示例：使用 STL 风格迭代器遍历 QVector

foreach 关键字

Qt 提供了 foreach 关键字（更推荐使用 const 版本 foreach (const T& t, container) {}），使得遍历容器更加简洁。这种方式基于 Qt 的元对象编译器（MOC）实现，底层实际上使用了迭代器，但开发者无需直接操作迭代器。

注意事项

  1. 动态删除

    • 在遍历容器时，删除元素会导致迭代器失效，从而引发未定义行为。可行的解决方案包括：
      • 在删除元素后重新获取迭代器。
      • 保存待删除的键或索引，待遍历结束后再删除。

  2. 选择合适的容器

    • QList: 适用于频繁的插入和删除操作。
    • QVector: 适用于实现随机访问为主的场景，类似于 std::vector。
    • QMap: 适用于快速查找的场景。
    • QSet: 适用于唯一元素集合的快速查找。
    • QLinkedList: 适用于频繁的插入和删除，但不需要随机访问。

  3. 只读访问

    • 使用只读迭代器（以 Const 开头的迭代器）可以确保数据的安全性，同时允许更多的线程安全访问。例如：

  4. 迭代器失效问题

    • 大多数容器操作（如插入、删除、排序）会使迭代器失效，导致 undefined behavior。应避免在修改容器时保存迭代器，或在容器变更后使用旧的迭代器。

选择合适的迭代方式

根据具体需求选择合适的迭代方式可以显著提高程序的效率和可维护性：

需求 建议的迭代方式 优点 缺点
简洁代码 foreach 关键字 简单、代码简洁 只能读取，无法修改内容
高效访问 STL 风格迭代器 最高效 代码相对复杂
底层控制 Java 风格迭代器 易于使用 效率不如 STL 迭代器
滑块专用 特定的容器迭代器 最好适合特定容器的特定需求 语法不统一
多线程环境 使用 const 引用 安全 适用性有限

通过合理选择容器和迭代器类型，开发者可以有效提高程序的效率和代码质量，并降低潜在的维护成本。

QVariant

由于 C++ 禁止 union 包含具有非默认构造函数或析构函数的类型，因此最有趣的 Qt 类不能用于联合。如果没有 QVariant, 这将成为 QObject::property () 和数据库工作等的问题。

QVariant 对象一次只包含一个 type () 的一个值。(一些 type () 是多值的，例如字符串列表。) 可以找出变体 T 所包含的类型，使用 convert () 将其转换为另一种类型，使用 toT () 函数 (例如 toSize ()) 获取其值，并使用 canConvert () 检查该类型是否可以转换为特定类型。

名为 toT () 的方法 (例如 toInt ()、toString ()) 是 const。如果您请求存储类型，它们将返回存储对象的副本。如果您请求一个可以从存储类型生成的类型，toT () 将复制并转换，并保持对象本身不变。如果您请求一个不能从存储类型生成的类型，结果取决于类型；详情请参阅函数文档。

连续使用 canConvert () 和 convert ()

当连续使用 canConvert () 和 convert () 时，canConvert () 可能返回 true, 而 convert () 可能返回 false。这通常是因为 canConvert () 只报告 QVariant 在给定适当数据的情况下在类型之间进行转换的一般能力；仍然可能提供实际上无法转换的数据。

例如，当调用包含字符串的变体时，canConvert (Int) 会返回 true, 因为原则上 QVariant 能够将数字字符串转换为整数。然而，如果字符串包含非数值字符，它就无法转换为整数，任何转换尝试都将失败。因此，为了成功转换，两个函数都返回 true 是非常重要的。

QMake 和 CMake 对比及实践指南

QMake 和 CMake 都是构建系统，负责将源代码转换为可执行程序。两者的核心作用相同：根据项目配置生成适用于特定编译器的 Makefile。

用一个类比来理解：构建系统类似于作曲家，Makefile 是乐谱，编译器则是演奏乐章的乐队。QMake 和 CMake 就是两位风格不同的作曲家——前者专注于 Qt 领域，后者通用性更强。

一、核心特性对比

1.1 适用场景

构建系统 最佳使用场景 典型项目类型
QMake 纯 Qt 项目，与 Qt Creator 深度集成 桌面 GUI 应用、Qt Quick 应用
CMake 跨平台混合项目，需要集成第三方库或多语言模块 大型工程、嵌入式系统、跨语言项目

设计动机说明：

  • QMake 由 Qt 团队开发，目标是简化 Qt 项目构建，.pro 文件语法与 Qt 模块命名高度一致
  • CMake 是通用构建系统，支持 C/C++/Fortran 等多种语言，Qt 从 5.x 开始官方支持 CMake，Qt 6 已将 CMake 作为推荐构建系统 [1]

1.2 配置复杂度

  • QMake：通过 .pro 文件定义项目，语法简洁，5 分钟即可上手
  • CMake：需要理解 CMakeLists.txt 语法、变量作用域、模块系统，完整掌握需要 1-2 周

1.3 社区与生态

维度 QMake CMake
官方支持 Qt 官方维护 Kitware 维护，Qt 官方适配
文档质量 Qt 文档集成，示例丰富 官方参考手册 + 大量社区教程
IDE 支持 Qt Creator 原生支持 CLion、VS Code、Visual Studio 原生支持
第三方库集成 需手动配置路径 通过 find_package 自动定位

[!TIP]
Qt 6 项目推荐使用 CMake。Qt 官方从 Qt 6 开始已将 CMake 作为默认构建系统，QMake 仅保留兼容支持 [1]。

五、IDE 集成

5.1 VS Code 集成

必需扩展

  1. **C/C++**：提供代码高亮、智能感知、调试支持
  2. CMake：CMakeLists.txt 语法高亮
  3. CMake Tools：项目配置、构建和调试集成

配置步骤

  1. 打开项目文件夹（包含 CMakeLists.txt）
  2. VS Code 自动检测 CMake 项目，选择 CMake Kit（推荐 Visual Studio 2019/2022）
  3. 按 F7 构建项目，按 Shift+F5 运行，按 Ctrl+F5 调试

Qt Creator 原生支持 CMake 项目：

  1. 选择 文件 → 打开文件或项目，选择 CMakeLists.txt
  2. 选择构建套件（Kit），点击 配置项目
  3. 使用标准的构建和运行按钮

七、选型建议

总结：

  1. 新项目：优先选择 CMake，尤其是计划使用 Qt 6 的项目
  2. 现有 QMake 项目：若无迁移压力可继续使用，升级 Qt 6 时再迁移
  3. 混合项目：必须使用 CMake，可通过 add_subdirectory 集成各模块

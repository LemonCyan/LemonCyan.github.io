QMake 和 CMake 对比及实践指南

QMake 和 CMake 都是用于控制编译过程的构建系统。它们的最终目标都是生成适用于特定编译器环境的本机 makefile 和工作区。简单来说，就像作曲和乐谱的关系，QMake 和 CMake 就像作曲者，makefile 是乐谱，而编译器则是演奏乐章的乐队。以下我们将详细探讨这两种构建工具的区别，并提供实践中的转换指南。

一、QMake 和 CMake 的对比

1. 适用场景

  • QMake：专为 Qt 项目设计，特别适合纯 Qt 项目。它与 Qt Creator 完美集成，通过 .pro 文件即可轻松配置项目，效率高、速度快。
  • CMake：功能更通用，适合复杂项目，尤其是需要跨语言构建的场景。例如，大型项目中包含 Qt 子工程、C++ 模块甚至第三方库时，CMake 的灵活性和扩展性更胜一筹。

2. 配置复杂度

  • QMake：配置简单，通过 .pro 文件直接定义项目需求。
  • CMake：配置灵活但学习曲线较高，需要理解 CMakeLists.txt 文件的语法和模块化机制。

3. 社区支持

  • QMake：Qt 官方支持，文档和社区资源丰富。
  • CMake：开源社区活跃，适用于广泛的构建场景，支持多种编译器和 IDE。

二、CMake 基础入门

CMake 通过 CMakeLists.txt 文件描述构建过程。以下是关键命令和用法：

1. 常用指令：

  • **ADD_SUBDIRECTORY**：递归地添加子目录，查找子目录中的 CMakeLists.txt。

  • **ADD_EXECUTABLE**：生成可执行文件。

  • **ADD_LIBRARY**：生成库文件（静态库或共享库）。

  • **FIND_LIBRARY**：查找系统中的库文件。

  • **SET**：定义变量。

  • **CMAKE_MODULE_PATH**：指定 CMake 模块的搜索路径。

2. 输出路径控制

  • EXECUTABLE_OUTPUT_PATH 和 **LIBRARY_OUTPUT_PATH**：通过 SET 指令控制目标文件的输出目录。

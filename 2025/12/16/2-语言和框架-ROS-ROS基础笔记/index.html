ROS基础笔记

基础概念

Nodes

节点，节点可以与同一进程/不同进程/不同设备上的节点进行通讯，节点之间的连接通过分布式发现过程 (distributed discovery process) 建立。

节点可以充当订阅者/发布者/服务 server/服务 client/action server/action client

Discovery

发现，通过 ROS2 的底层中间件自动完成的。 稳定的连接依赖于 Quality of Service

  • 启动时，向具有相同 ROS_DOMAIN_ID 的其他节点广播
  • 定期广播自身存在
  • 离线警报

Interface

接口，通常使用三种类型的接口进行通讯：Topic/Service/Action

  • .msg 描述消息的各个字段
  • .srv 描述服务，请求和响应组成
  • .action 描述操作，由目标 goal，结果 result 和反馈 feedback 组成

Topic

节点间发送数据的一种方式，其他节点可以订阅 Topic 并接收消息。.msg 文件由两部分组成，field 和 constant

  • 任何特定主题可以有零个或多个发布者和零个或多个订阅者
  • 匿名的
  • 强类型的

filed

  • 定义方式 fieldtype1 fieldname1 (defaultvalue)
  • 内置数据类型可以定义数组
  • 默认值

constant

  • 常量名必须大写 constanttype CONSTANTNAME=constantvalue

Service

  • 客户端等待服务器进行简单的计算并返回结果，需要快速响应
  • .msg 文件由请求和响应消息类型组成，两者之间用 --- 分割
  • 任何两个以 --- 连接的 .msg 文件均构成合法的服务描述。
  • service 通过 service name 识别
  • 每一个 service name 只能对应一个 service server，可以对应任意数量的 service client

Action

  • Action 是一种耗时的请求/响应通讯
  • Action 可以长时间运行，执行过程中提供反馈，并且可以被中断
  • eg 机器人的高级状态机可能会调用一个动作来指示导航子系统前往某个航点，导航子系统可以提供当前行进距离的反馈，而高级状态机可以选择取消或抢占前往该航点的路线

Parameter

  • 参数的声明周期和节点的生命周期绑定

  • 参数寻址方式：节点名称/节点命名空间/参数名称/参数命名空间

  • 参数由键值和描述符组成

  • 用于配置算法、硬件参数、行为开关等

  • 参数声明 默认情况下，节点声明其生命周期内的所有参数。

  • 参数类型

  • 参数回调 可以注册三种不同类型的回调函数，在参数发生变化时进行通知

  • 参数交互

  • 参数初始值设定

    • 运行节点时通过命令行参数或 yaml 文件设定初始参数值
    • 启动节点时通过启动工具设置参数初始值

  • 节点运行时操作参数值 ros2 param

ROS2 命令行工具

The source code for the ros2 command is available at https://github.com/ros2/ros2cli.

  • action: Introspect/interact with ROS actions 检查/交互 ROS 动作
  • bag: Record/play a rosbag 记录/播放 rosbag
  • component: Manage component containers 管理组件容器
  • daemon: Introspect/configure the ROS 2 daemon 检查/配置 ROS2 守护进程
  • doctor: Check ROS setup for potential issues 检查 ROS 配置潜在问题
  • interface: Show information about ROS interfaces 显示 ROS 接口信息
  • launch: Run/introspect a launch file 运行/检查启动文件
  • lifecycle: Introspect/manage nodes with managed lifecycles 检查/管理受控生命周期节点
  • multicast: Multicast debugging commands 多播调试命令
  • node: Introspect ROS nodes 检查 ROS 节点
  • param: Introspect/configure parameters on a node 检查/配置节点参数
  • pkg: Introspect ROS packages 检查 ROS 软件包
  • plugin: Introspect ROS plugins 检查 ROS 插件
  • run: Run ROS nodes 运行 ROS 节点
  • security: Configure security settings 配置安全设置
  • service: Introspect/call ROS services 检查/调用 ROS 服务
  • test: Run a ROS launch test 运行 ROS 启动测试
  • topic: Introspect/publish ROS topics 检查/发布 ROS 主题
  • trace: Tracing tools to get information on ROS nodes execution (only available on Linux)
  • wtf: An alias for doctor doctor 别名

ROS 守护进程

  • 守护进程使用 127.0.0.1 进行通讯，使用 ROS_DOMAIN_ID 环境变量作为端口号偏移量
  • 不同的 ROS_DOAMIN_ID 导致不同的守护实例运行在不同的端口上

启动

  • ROS2 由许多节点组成，节点运行在不同的进程/机器上
  • launch system 旨在通过单个命令运行多个节点
  • 描述系统配置，然后按描述启动节点
    • 系统配置包括要运行的程序，运行位置，传递参数
  • 负责监控已启动进程状态，报告和响应进程状态变化

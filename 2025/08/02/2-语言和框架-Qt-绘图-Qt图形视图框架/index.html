Qt图形视图框架

图形视图框架

图形视图提供了一个用于管理和与大量自定义 2D 图形项目交互的表面，以及一个用于可视化项目的视图小部件，并支持缩放和旋转。

该框架包含一个事件传播架构，允许场景中的物体具有精确的双精度交互能力。物体可以处理关键事件、鼠标点击、移动、释放和双击事件，它们还可以跟踪鼠标的移动。

Graphic View 使用二分空间划分 (BSP) 树来提供非常快速的项目发现，因此，它可以实时可视化大型场景，即使有数百万个项目。

图形视图是在 Qt 4.2 中引入的，取代了其前身 QCanvas。

图形视图架构

图形视图提供了一种基于项目的模型视图编程方法，很像 InterView 的方便类 QTableView、QTreeView 和 QListView。多个视图可以观察单个场景，该场景包含不同 Unicode 几何形状列表的项目。

场景

QGraphicsScene 提供了图形视图场景。该场景具有以下职责：

  • 为管理大量项目提供快速接口
  • 将事件传播到每个项目
  • 管理 item 状态，如选择和焦点处理
  • 提供未转换的渲染功能；主要用于打印
    场景作为 QGraphicsItem 对象的容器。项目通过调用 QGraphicsScene:：addItem () 添加到场景中，然后通过调用众多项目发现函数之一进行检索。QGraphicsScene:：items () 及其重载返回与点、矩形、多边形或一般向量路径相交或包含的所有项目。QGraphicsScene:：itemAt () 返回特定点上最高的项目。所有项目发现函数都以降序堆叠顺序返回项目 (即，第一个返回的项目是最高的，最后一个返回的项目是最低的)。

QGraphicsScene 的事件传播架构调度场景事件以传递到项目，并管理项目之间的传播。如果场景在特定位置接收到鼠标点击事件，场景会将该事件传递给该位置的任何项目。

QGraphicsScene 还管理某些项目状态，例如项目选择和焦点。可以通过调用 QGraphicsScene:：setSelectionArea (), 传递任意形状来选择场景中的项目。这个功能也用作 QGraphicsView 中橡皮筋选择的基础。要获取当前选定项目的列表，请调用 QGraphicsScene:：selectedItems ()。QGraphicsScene 处理的另一个状态是项目是否具有键盘输入焦点。可以通过调用 QGraficsScene:：setFocusItem () 或 QGraphicsItem:：setFocus () 来设置项目的焦点，或者通过调用 QGraphicsScene: :focusItem () 来获取当前焦点项目。

最后，QGraphicsScene 允许您通过 QGraphicsScene:：render () 函数将场景的部分渲染到绘图设备中。您可以在本文档后面的 “打印” 部分中阅读更多相关内容。

视图

QGraphicsView 提供了视图控件，它可视化场景的内容。可以将多个视图附加到同一场景中，以便在同一数据集中提供多个视口。视图控件是一个滚动区域，为浏览大型场景提供滚动条。要启用 OpenGL 支持，可以通过调用 QGraphicsView:：setViewport () 将 QGLWidget 设置为视口。

视图从键盘和鼠标接收输入事件，并将这些事件转换为场景事件 (在适当的情况下将使用的坐标转换为场景坐标), 然后将事件发送到可视化场景。

使用其变换矩阵 QGraphicsView:：transform (), 视图可以变换场景的坐标系。这允许高级导航功能，如缩放和旋转。为了方便，QGraphicsView 还提供了视图和场景坐标之间的翻译函数：QGraphicsView:：mapToScene () 和 QGraphicsView:：mapFromScene ()。

项目

QGraphicsItem 是场景中图形项目的基类。图形视图为典型形状提供了几个标准项目，例如矩形 (QGraphicsRectItem)、椭圆形 (QGraphicsEllipseItem) 和文本项目 (QGraphicsTextItem), 但最强大的 QGraphicsItem 功能在您编写自定义项目时才可用。除了其他功能外，QGraphicsItem 还支持以下功能：

  • 鼠标按下、移动、释放和双击事件，以及鼠标悬停事件、轮事件和上下文菜单事件。
  • 键盘输入焦点和关键事件
  • 拖放
  • 分组，无论是通过亲子关系还是使用 QGraphicsItemGroup
  • 碰撞侦测
    项目生活在局部坐标系中，像 QGraphicsView 一样，它也提供了许多功能来映射项目与场景之间以及项目与项目之间的坐标系。此外，像 QGraphicsView 一样，它可以使用一个矩阵来转换其坐标系：QGraphicsItem:：transform ()。这对于旋转和缩放单个项目很有用。
    元素可以包含其他元素 (子元素)。父元素的变换会被其所有子元素继承。然而，无论元素的累积变换如何，其所有函数 (例如，QGraphicsItem:：contains ()、QGraphicsItem:：boundingRect ()、QGraphicsItem:：collidesWith ()) 仍然在局部坐标中运行。
    QGraphicsItem 通过 QGraphicsItem:：shape () 函数和 QGraphicsItem:：collidesWith () 函数支持碰撞侦测，这两个函数都是虚拟函数。通过从 QGraphicsItem:：shape () 返回项目的形状作为局部坐标 QPainterPath,QGraphicsItem 将为你处理所有碰撞侦测。然而，如果你想提供自己的碰撞侦测，可以重新实现 QGraphicsItem:：collideswith ()。

图形视图坐标系

图形视图基于笛卡儿坐标系；场景中物体的位置和几何形状由两个数字集表示：x 坐标和 y 坐标。当使用未变换视图观察场景时，场景中的一个单元由屏幕上的一个像素表示。

注意：由于图形视图使用 Qt 的坐标系，因此不支持倒 Y 轴坐标系 (其中 y 向上增长)。

图形视图中有三个有效的坐标系统：项目坐标、场景坐标和视图坐标。为了简化实现，图形视图提供了便利的功能，允许您在三个坐标系之间进行映射。

渲染时，图形视图的场景坐标对应于 QPainter 的逻辑坐标，视图坐标与设备坐标相同。在坐标系文档中，您可以了解逻辑坐标与设备坐标之间的关系。

项目坐标

物品生活在自己的局部坐标系中。它们的坐标通常以其中心点 (0,0) 为中心，这也是所有变换的中心。物品坐标系中的几何基元通常被称为物品点、物品线或物品矩形。

创建自定义项目时，项目坐标是你唯一需要考虑的；QGraphicsScene 和 QGraphicsView 将为你执行所有转换。这使得实现自定义项目非常容易。例如，如果你收到鼠标点击或拖动输入事件，事件位置将在项目坐标中给出。QGraphicsItem:：contains () 虚拟函数在项目坐标中接受一个 point 参数，如果某个点位于项目内部，则返回 true, 否则返回 false。类似地，项目的边界直线和形状也位于项目坐标中。

项目的位置是其父坐标系中项目中心点的坐标系，有时也称为父坐标系。从这个意义上说，场景被视为所有无父项目的 “父”。顶级项目的位置在场景坐标系中。

子坐标相对于父坐标。如果子坐标未变换，则子坐标和父坐标之间的差值等于父坐标中项之间的距离。例如：如果未变换的子项精确位于其父坐标的中心点，则两个项的坐标系将完全相同。然而，如果子坐标为 (10,0), 则子坐标的 (0,10) 点将对应于其父坐标的 (10,10) 点。

由于项目的位置和变换相对于父项，因此子项的坐标不受父项变换的影响，尽管父项的变换隐式地变换了子项。在上述例子中，即使父项旋转并缩放，子项的 (0,10) 点仍然对应于父项的 (10,10) 点。然而，相对于场景，子项将遵循父项的变换和位置。如果父项缩放 (2x,2x), 子项的位置将位于场景坐标 (20,0), 其 (10,0) 点将对应于场景中的 (40,0) 点。

QGraphicsItem:：pos () 是少数例外之一，QGraphicsItem 的函数在项目坐标中运行，而不考虑项目或其父项的任何变换。例如，项目的边界 rect (即 QGraphicsItem:：boundingRect ()) 始终在项目坐标中给出。

场景坐标

场景代表其所有项目的基本坐标系。场景坐标系描述每个顶级项目的位置，并为从视图传递到场景的所有场景事件提供基础。场景中的每个项目都有一个场景位置和边界矩形 (QGraphicsItem:：scenePos ()、QGraphicsItem:：sceneBoundingRect ()), 以及局部项目的位置和边界矩形。场景位置描述项目在场景坐标中的位置，其场景边界矩形构成 QGraphicsScene 如何确定场景哪些区域发生变化的基础。场景中的变化通过 QGraphicsScene:：changed () 信号传达，参数是场景矩形的列表。

视图坐标

视图坐标系是控件的坐标。视图坐标系中的每个单元对应一个像素。这个坐标系的特殊之处在于它相对于控件或视口，不受所观察场景的影响。QGraphicsView 视口的左上角始终为 (0,0), 右下角始终为 (视口宽度，视口高度)。所有鼠标事件和拖放事件最初都作为视图坐标接收，您需要将这些坐标映射到场景中才能与项目交互。

坐标映射

通常，在处理场景中的项目时，将坐标和任意形状从场景映射到项目、从项目映射到项目，或者从视图映射到场景是很有用的。例如，当你在 QGraphicsView 的视口中单击鼠标时，可以通过调用 QGraphicsView:：mapToScene (), 然后调用 QGraphicsScene:：itemAt () 来询问场景光标下的项目是什么。如果你想知道项目在视口中的位置，可以在项目上调用 QGraphicsItem:：mapToScene (), 然后在视图上调用 QGraphicsView:：mapFromScene ()。最后，如果你使用了 “查找视图椭圆内的项目”, 可以将 QPainterPath 传递给 mapToScene (), 然后将映射的路径传递给 QGraphicsScene:：items ()。

你可以通过调用 QGraphicsItem:：mapToScene () 和 QGraphicsItem:：mapFromScene () 将坐标和形状映射到项目的场景中或从场景中映射出来。你还可以通过调用 QGraphicsItme:：mapToParent () 和 QGraphicsItem:：mapfromParent () 映射到项目的父项，或者通过调用 QGraphicsItem: :mapToItem () 和 QGraphicsItem:：map FromScene () 在项目之间进行映射。所有映射函数都可以映射点、矩形、多边形和路径。

视图中提供了相同的映射函数，用于映射到场景和从场景映射。QGraphicsView:：mapFromScene () 和 QGraphicsView:：mapToScene ()。要从视图映射到项目，首先映射到场景，然后从场景映射到项目。

当源区域和目标区域的大小不匹配时，源内容会被拉伸以适合目标区域。通过将 Qt:：AspectRatioMode 传递给正在使用的渲染函数，您可以选择在内容被拉伸时维护或忽略场景的宽高比。

拖放

动画

图形视图支持多个级别的动画。您可以使用动画框架轻松组装动画。为此，您需要从 QGraphicsObject 继承您的项目，并将 QPropertyAnimation 与它们关联起来。QPropertyAnimation 允许对任何 QObject 属性进行动画处理。

另一个选择是创建一个继承自 QObject 和 QGraphicsItem 的自定义项目。该项目可以设置自己的定时器，并通过 QObject:：timerEvent () 中的增量步骤来控制动画。

第三个选项主要用于 Qt 3 中 QCanvas 的兼容性，即通过调用 QGraphicsScene:：advance () 来推进场景，这反过来又调用 QGraphicsItem:：advance ()。

项目分组

通过使一个物品成为另一个物品的子物品，你可以实现物品分组最基本的特征：物品将一起移动，所有的变换都会从父母传递给子物品。

此外，QGraphicsItemGroup 是一个特殊的项目，它结合了子事件处理和一个有用的接口，用于在组中添加和删除项目。向 QGraphicsItemGroup 添加项目将保留项目的原始位置和变换，而通常重新父化项目将导致子项相对于其新父项重新定位自己。为了方便起见，可以通过调用 QGraphicsScene:：createItemGroup () 来创建 QGraphicsItemGroups。

小部件和布局

Qt 4.4 通过 QGraphicsWidget 引入了对几何形状和布局感知项的支持。这个特殊的基础项类似于 QWidget, 但与 QWidget 不同的是，它不是继承自 QPaintDevice, 而是来自 QGraphicsItem。这允许你编写包含事件、信号和槽、大小提示和策略的完整构件，你还可以通过 QGraphicsLinearLayout 和 QGraphicsGridLayout 管理构件在布局中的几何形状。

QGraphicsWidget

在 QGraphicsItem 的功能和精简版本的基础上，QGraphicsWidget 提供了两全其美的优势：来自 QGraphicsWidget 的额外功能，如样式、字体、调色板、布局方向及其几何形状，以及来自 QGraphicsItem 的分辨率独立性和转换支持。由于图形视图使用实际坐标而不是整数，QGraphicsWidget 的几何函数也可以在 QRectF 和 QPointF 上运行。这也适用于帧直线、外边距和间距。例如，在 QGraphicsWidget 中，指定 (0.5、0.5、0.5、0.5) 的内容外边距并不罕见。你可以创建子控件和 “顶级” 窗口；在某些情况下，你现在可以使用图形视图进行高级 MDI 应用程序。

支持 QGraphicsWidget 的一些属性，包括窗口标志和属性，但并非全部。你应该参考 QGraphicsWidget 的类文档，了解哪些属性受支持、哪些属性不受支持的完整概述。例如，你可以通过将 Qt:：Window 窗口标志传递给 QGraphicsWidget 的构造函数来创建装饰窗口，但图形视图目前不支持 macOS 上常见的 Qt:：Sheet 和 Qt:：Drawer 标志。

QGraphicsLayout

QGraphicsLayout 是专门为 QGraphicsWidget 设计的第二代布局框架的一部分。它的 API 与 QLayout 非常相似。你可以在 QGraphicsLinearLayout 和 QGraphicsGridLayout 中管理构件和子布局。你还可以通过自己对 QGraphicsLayout 进行子类化来轻松编写自己的布局，或者通过编写 QGraphicsLayoutItem 的适配器子类来将自己的 QGraphicsItem 项添加到布局中。

嵌入式控件支持

图形视图为将任何构件嵌入场景提供无缝支持。您可以嵌入简单的构件，如 QLineEdit 或 QPushButton, 也可以嵌入复杂的构件，如 QTabWidget, 甚至可以完成主窗口。要将构件嵌入场景，只需调用 QGraphicsScene:：addWidget (), 或者创建 QGraphicsProxyWidget 的实例来手动嵌入构件。

通过 QGraphicsProxyWidget,GraphicsView 能够深度集成客户端构件功能，包括其光标、工具提示、鼠标、平板电脑和键盘事件、子构件、动画、弹出窗口 (例如 QComboBox 或 QCompleter), 以及构件的输入焦点和激活。QGraphicsProxyWidget 甚至集成了嵌入式构件的选项卡顺序，因此您可以在嵌入式构件中进行选择和取消选择。您甚至可以在场景中嵌入一个新的 QGraphicsView, 以提供复杂的嵌套场景。

在转换嵌入式小部件时，Graphics View 确保小部件独立转换分辨率，使字体和样式在放大时保持清晰。(请注意，分辨率独立性的影响取决于样式。)

性能

浮点说明

为了准确快速地将变换和效果应用于项目，Graphics View 的构建假设是用户的硬件能够为浮点指令提供合理的性能。

许多工作站和台式计算机都配备了适当的硬件来加速这种计算，但一些嵌入式设备可能只提供用于处理数学运算或在软件中模拟浮点指令的库。

因此，某些类型的效果在某些设备上可能比预期慢。可以通过在其他领域进行优化来弥补这种性能影响；例如，使用 OpenGL 渲染场景。然而，如果这些优化也依赖于浮点硬件的存在，它们本身可能会导致性能下降。

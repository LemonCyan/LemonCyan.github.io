读写函数

读写函数分类

三类读写函数

  • 行缓存函数：遇到新行符 (\n) 或写满缓存时调用系统调用函数
    • 读函数：fgets, getc, getchar, fscanf
    • 写函数：fputs, puts, printf
  • 无缓存函数：用户调用函数时直接将内容写入内核
  • 全缓存函数：写满缓存后调用系统调用函数
    • 读函数：fread
    • 写函数：fwrite

行缓存读写函数

fgets 函数

  • 原型：char* fgets(char* s, int size, FILE *stream)
  • 参数：
    • s：缓存地址，用于存储读取的内容
    • size：读取的字节数
    • stream：文件流
  • 返回值：成功返回缓存地址 s，文件结尾或出错返回 NULL
  • 功能：从文件流中读取字符串到缓存，最多读取 size-1 个字符，会保留换行符

fputs 函数

  • 原型：int fputs(const char* s, FILE *stream)
  • 参数：
    • s：要写入的字符串
    • stream：文件流
  • 返回值：成功返回非负值，出错返回 EOF(-1)
  • 功能：将字符串写入文件流，不自动添加换行符，关闭文件前会刷新缓存

fclose 函数

  • 原型：int fclose(FILE *stream)
  • 返回值：成功返回 0，失败返回 EOF 并设置 errno
  • 功能：关闭文件流，刷新缓存并释放自动分配的缓存

fread 和 fwrite 函数

刷新缓存函数

fflush 函数

  • 原型：int fflush(FILE *stream)
  • 功能：强制将库函数缓存中的内容写入内核
  • 应用场景：需要立即写入数据而不等待缓存条件满足时

调整读写位置指针函数

fseek 函数

  • 原型：int fseek(FILE *stream, long offset, int whence)
  • 参数：
    • offset：偏移量
    • whence：起始位置（SEEK_SET、SEEK_CUR、SEEK_END）
  • 返回值：成功返回 0，失败返回 -1
  • 功能：设置文件位置指针，与 lseek 参数相同但返回值不同

rewind 函数

  • 原型：void rewind(FILE *stream)
  • 功能：将文件位置指针重置到文件开头
  • 等价于：(void)fseek(stream, 0, SEEK_SET)

ftell 函数

  • 原型：long ftell(FILE *stream)
  • 返回值：返回当前文件位置指针值
  • 功能：获取文件位置指针的当前位置

gets 和 puts 函数

gets 函数

  • 原型：char* gets(char *s)
  • 风险：无法指定缓存长度，可能导致缓冲区溢出
  • 限制：只能从标准输入读取，不存储换行符

puts 函数

  • 原型：int puts(const char *s)
  • 特点：只能向标准输出写入，输出时自动添加换行符

与 fgets/fputs 的区别

函数 输入 / 输出范围 换行符处理 缓存长度限制
gets 标准输入 不存储换行符 无
fgets 任意文件流 存储换行符 有
puts 标准输出 自动添加换行符 无
fputs 任意文件流 不添加换行符，原样输出 有

单字符读写函数

fgetc 函数

  • 原型：int fgetc(FILE *stream)
  • 返回值：成功返回读取的字符，文件结尾或出错返回 EOF
  • 功能：从文件流中读取一个字符

fputc 函数

  • 原型：int fputc(int c, FILE *stream)
  • 返回值：成功返回写入的字符，出错返回 EOF
  • 功能：向文件流中写入一个字符
  • 缓存特性：非行缓存函数，需手动调用 fflush 或 fclose 刷新

文件状态判断函数

feof 函数

  • 原型：int feof(FILE *stream)
  • 返回值：文件结束返回非 0，否则返回 0
  • 功能：判断是否到达文件结尾

ferror 函数

  • 原型：int ferror(FILE *stream)
  • 返回值：读写错误返回非 0，否则返回 0
  • 功能：判断是否发生读写错误。检测流上的错误。即：检查文件在使用各种输入输出函数进行读写时是否出错。当输入输出函数对文 件进行读写时出错，文件就会产生错误标志。应用这个函数，就可以检查出 fp 所指向的文件操作是否 出错，也就是说是否有错误标志

clearerr 函数

  • 原型：void clearerr(FILE *stream)
  • 功能：清除文件流的错误状态。复位错误标志，即：使 fp 所指向的文件中的错误标志和文件结束标志置 0。当输入输出函数对文件进行 读写出错时，文件就会自动产生错误标志，这样会影响程序对文件的后续操作。clearerr 函数就是要复 位这些错误标志，也就是使 fp 所指向的文件的错误标志和文件结束标志置 0，从而使文件恢复正常。

安全注意事项

  • gets 函数风险：由于无法限制输入长度，可能导致缓冲区溢出，建议使用 fgets 替代
  • 缓存刷新时机：若未写满缓存且无换行符，数据不会自动写入文件，需手动调用 fflush 或确保程序正常关闭以触发 fclose 刷新

fopen（打开文件）

相关函数: open, fclose
表头文件: #include <stdio.h>

函数定义:

函数说明:
fopen() 用于打开文件并返回一个文件指针。

参数说明:

  • path: 包含文件路径及文件名的字符串
  • mode: 指定文件的打开模式（见下表）

文件打开模式:

模式字符串 描述
r 以只读方式打开文件，文件必须存在。
r+ 以可读写方式打开文件，文件必须存在。
w 以只写方式打开文件。如果文件存在，则长度清为零；若不存在，则创建新文件。
w+ 以可读写方式打开文件。如果文件存在，长度清为零；若不存在，则创建新文件。
a 以只写方式打开文件，且在文件末尾追加内容。如果文件不存在，则创建新文件。
a+ 以可读写方式打开文件，且在文件末尾追加内容。如果文件不存在，则创建新文件。

模式字符串的扩展:
以上模式字符串可以添加 b 字符（如 rb, w+b 等），以指定打开二进制文件。不过在 POSIX 系统（包括 Linux）中，这个字符会被忽略。

文件权限:
使用 fopen() 打开的新文件默认权限为 0666（即 S_IRUSR|S_IWUSR|S_IRGRP|S_IWGRP|S_IROTH|S_IWOTH），实际权限会参考当前 umask 值。

返回值:

  • 成功：返回文件指针 FILE *
  • 失败：返回 NULL，错误代码存入 errno

附加说明:

  • 打开文件后应立即进行错误检查，因开文件失败会导致后续读写操作无法进行。
  • 文件指针用于后续的读写操作，需在关闭文件前确保所有缓冲区数据已刷新到磁盘。

flock (文件锁定/解锁)

flock 函数用于在整个文件上应用或移除锁定。与 fcntl 不同，它不支持基于区域的锁定。

包含 flock 所需的头文件：

  • fd: 文件描述符。

  • operation: 锁操作类型。

  • LOCK_SH: 共享锁，允许多个进程进行读取。

  • LOCK_EX: 排他锁，只允许一个进程进行写入。

  • LOCK_UN: 释放锁定。

  • LOCK_NB: 非阻塞；如果锁定失败，立即返回。

  • 0 表示成功；-1 表示失败，并设置 errno。

注意:慎用此函数，因不当锁定可能导致竞态条件。

sscanf：从缓冲区中读取格式化字符串函数

  1. 头文件
  1. 函数原型
  1. 函数功能
    sscanf 函数从 string 指定的缓冲区中读取格式化数据，并将其存入 argument 指向的变量中。format 指定了数据的格式，而 argument 必须是相应类型的指针。

需要注意的是，sscanf 与 scanf 和 fscanf 的不同之处在于：

  • sscanf 从内存缓冲区读取数据。
  • scanf 从标准输入（stdin）读取数据。
  • fscanf 从指定文件读取数据。
  1. 返回值
    sscanf 返回成功匹配并存储的项目数。返回 0 表示没有匹配到任何数据，返回 EOF 表示读取失败。

注意：如果输入数据与 format 不匹配，可能会导致数据存储在错误的位置，甚至导致程序崩溃。

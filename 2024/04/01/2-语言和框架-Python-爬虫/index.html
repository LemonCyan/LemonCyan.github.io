爬虫

前言

最近想把一个博客网站的文章都爬下来存到本地，正好借这个机会记录一下爬虫的设计思路。这篇笔记会从零开始讲解，即使你没有太多编程经验，跟着做也能写出自己的爬虫。

工具选择

爬虫说白了就是模拟浏览器去访问网页，然后把需要的内容提取出来。Python 有很多现成的库可以用：

工具 用途 特点
requests 发送 HTTP 请求 简单好用，大多数情况够用
BeautifulSoup 解析 HTML 语法直观，容易上手
Selenium 浏览器自动化 能处理 JavaScript 渲染的页面，但速度慢
Scrapy 专业爬虫框架 功能强大，适合大型项目

对于普通的静态网站，requests + BeautifulSoup 就足够了。如果是那种需要 JavaScript 才能加载内容的网站（比如无限滚动加载的页面），就需要用 Selenium。

动手之前先搞清楚网站结构

不管爬什么网站，第一步都是打开浏览器的开发者工具（F12），看看网页的真实结构。

查看网页源码

右键 -> 查看网页源码，看看内容是直接写在 HTML 里，还是通过 JavaScript 动态加载的。

分析数据来源

切换到 Network 标签，刷新页面，观察网络请求。很多现代网站的数据是通过 API 返回的 JSON 格式，直接请求 API 比解析 HTML 方便多了。

找规律

比如文章列表页的 URL 是不是有规律？分页是怎么实现的？文章详情页的 URL 格式是什么样的？这些规律决定了爬虫的设计思路。

实战：爬取 findmoney.club 博客

下面以 https://www.findmoney.club/ 为例，演示完整的爬虫设计过程。

分析网站

这个网站用的是 Next.js 框架，有个特点：页面数据都嵌在 HTML 里的一个 <script id="__NEXT_DATA__"> 标签中，是 JSON 格式。

打开首页的源码，搜索 __NEXT_DATA__，你会发现里面有个 allNavPages 数组，包含了所有文章的基本信息：标题、分类、链接等。

同样地，打开某篇文章的页面，__NEXT_DATA__ 里有个 post 对象，包含文章的详细内容。这些数据存储在 Notion 的 block 结构中。

  • beautifulsoup4：解析 HTML（虽然我们主要解析 JSON，但也可能用到）

几个关键点

模拟浏览器

很多网站会检查 User-Agent 请求头，如果发现不是正常浏览器就会拒绝访问。所以我们要伪装成浏览器。

错误处理和重试

网络请求难免会失败，加上重试机制能提高成功率。

礼貌性延迟

不要一股脑地狂发请求，给对方服务器留点喘息的机会。这是爬虫的基本素养。

总结

设计爬虫的一般流程：

  1. 分析目标网站：用浏览器开发者工具研究页面结构和数据来源
  2. 选择合适的工具：静态页面用 requests，动态页面用 Selenium
  3. 编写爬虫逻辑：获取页面 -> 解析数据 -> 保存结果
  4. 处理异常情况：重试机制、错误处理、延迟控制

记住几个原则：

  • 遵守网站的 robots.txt 规则
  • 控制请求频率，不要给服务器造成压力
  • 仅用于个人学习和研究，不要用于商业用途或侵犯版权

爬虫本身是个中性的技术，关键看你怎么用它。希望这篇笔记能帮到你！

Posted on 2024-04-18 Edited on 2024-04-22

术语定义

OpenGL

全称OpenSource Graphic Library，是由Khronos Group维护的一个跨平台的图形API（应用程序接口），项目地址提供了OpenGL接口的标准/规范/头文件，从而一个标准化的方法来与GPU进行交互，使开发者能够跨不同的硬件和操作系统实现一致的图形渲染体验，确保API的跨平台兼容性和功能一致性。

OpenGL存在以下几个子集：

OpenGL ES 1.0/2.0/3.0：OpenGL for Embedded Systems的缩写，这是一个面向嵌入式系统（如移动设备、物联网设备等）的图形API。OpenGL ES与标准OpenGL相比进行了裁剪和优化，以适应移动和嵌入式设备的资源和性能限制。

WebGL：Web Graphics Library的缩写，这是一个基于OpenGL ES的Web图形API。WebGL允许开发者在Web浏览器中直接渲染3D和2D图形，为网页提供更丰富的视觉效果。

OpenGL SC 1.0/2.0

OpenGL Extensions：除了官方子集之外，OpenGL还支持各种扩展（extensions），这些扩展由硬件厂商或Khronos Group定义，提供额外的功能或性能优化。这些扩展可以是特定于某个硬件厂商或通用。

OpenGL SC

全称OpenGL Safety Critical，详细介绍，标准规范。一个专门针对安全关键系统（如航空、汽车、工业自动化等领域）开发的OpenGL子集。OpenGL SC旨在提供更高的可靠性和稳定性，以满足这些领域的严格要求。

精简OpenGL标准的功能，只保留了最基本和最常用的特性。降低系统的复杂性，提高稳定性和可靠性。

通过限制功能和明确规范，提供了可预测的行为。这在安全关键系统中至关重要，因为不确定性可能导致意外行为。

规范相对稳定，不会频繁更新。

强调资源的有效管理，如内存、处理器和图形资源，以避免出现资源争用或过度使用的问题。

系统安全性，包括避免潜在的安全漏洞和确保数据的完整性。

RTOS

RTOS的主要特征包括：

  • 实时性能： RTOS确保在特定时间框架内完成任务。它提供可预测和有保障的响应时间。
  • 任务调度： RTOS根据任务的优先级和紧急程度来管理任务。它使用调度算法有效地管理任务的执行。
  • 并发和多任务处理： RTOS允许多个任务同时运行，每个任务都有自己的上下文和状态。
  • 确定性： RTOS提供可预测和可重复的行为，确保任务每次执行都保持一致。
  • 低延迟： RTOS尽量减少事件发生和系统响应之间的时间。
  • 中断处理： RTOS能够快速、高效地处理硬件中断，确保及时执行任务。
  • 资源管理： RTOS有效管理系统资源，如CPU时间、内存和I/O设备，以优化性能。

ANGLE

ANGLE 的目标是通过将 OpenGL ES API 调用转换为该平台可用的硬件支持的 API 之一，允许多个操作系统的用户无缝运行 WebGL 和其他 OpenGL ES 内容。 ANGLE 目前提供从 OpenGL ES 2.0、3.0 和 3.1 到 Vulkan、桌面 OpenGL、OpenGL ES、Direct3D 9 和 Direct3D 11 的翻译。未来的计划包括 ES 3.2、到 Metal 和 MacOS、Chrome OS 和 Fuchsia 支持的翻译。

Mesa

Mesa 项目最初是 OpenGL 规范的开源实现 - 一个用于渲染交互式 3D 图形的系统。

多年来，该项目不断发展，实现了更多图形 API，包括 OpenGL ES、OpenCL、OpenMAX、VDPAU、VA-API、Vulkan 和 EGL。

各种设备驱动程序允许 Mesa 库在许多不同的环境中使用，从软件模拟到现代 GPU 的完整硬件加速。

Mesa 与其他几个开源项目相结合：直接渲染基础设施、X.org 和 Wayland，在 Linux、FreeBSD 和其他操作系统上提供 OpenGL 支持。

其依赖 libdrm,libxcb(libxcb依赖 x11,xcb-proto),libudev，移植参考文章

TinyGL

TinyGL 的一个分支实现 PicoGL。

PicoGL基于TinyGL 4.0，增加了直接写Linux Framebuffer的backend、使用Makefile组织项目、增加了定点数运算支持。要使用这个软件包还必须有支持软件浮点的交叉编译器。源码地址

标准文件

  • ARINC 653（Aeronautical Radio, Incorporated 653）是一种用于航空航天领域的实时操作系统（RTOS）标准
  • ARINC 661（Cockpit Display System Interface to User Systems）是一种航空电子标准，定义了用户系统与飞机座舱显示系统（Cockpit Display System，CDS）之间的接口。这个标准的目标是确保用户系统和显示系统之间的兼容性和互操作性，从而在飞机的座舱中提供安全、可靠和高效的操作环境。
  • DO-178C是航空航天领域中广泛使用的标准之一，用于指导安全关键航空软件的开发、测试和认证。DO-178C的全称是《软件在航空系统和设备认证中的考虑》（Software Considerations in Airborne Systems and Equipment Certification）。该标准由航空电子设备标准化组织（RTCA）和欧洲航空电子设备委员会（EUROCAE）共同制定。
  • EASA ED-12C是由欧洲航空安全局（EASA）发布的航空软件认证标准。ED-12C和DO-178C实际上是同一份标准，由RTCA和EUROCAE两家机构联合制定，并分别由RTCA发布为DO-178C，由EUROCAE发布为ED-12C。两者的内容完全相同，只是不同的命名方式。

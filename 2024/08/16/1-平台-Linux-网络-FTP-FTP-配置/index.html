FTP 配置

FTP 的数据传输模式

FTP（File Transfer Protocol）是一个基于客户端/服务器架构的文件传输协议，支持两种工作模式：

  • 主动模式：客户端向 FTP 服务器发送端口信息，服务器主动连接该端口。
  • 被动模式：FTP 服务器开启并发送端口信息给客户端，客户端连接服务器的端口，服务器被动接受连接。

说明：大多数 FTP 客户端运行在局域网中，通常没有独立的公网 IP 地址，并且可能会被防火墙拦截。在主动模式下，FTP 服务器连接客户端将面临更多困难。因此，除非有特殊需求，否则建议将 FTP 服务器配置为被动模式。

Linux 环境下的 FTP 服务器

在 Linux 环境下，有几种常用的 FTP 服务器软件，每种都有其特点和用途：

  • Wu-ftpd

    • 特点：Wu-ftpd 是历史最久的非商业 FTP 服务器之一，功能丰富，支持多种传输模式和用户管理方式。
    • 官网：http://www.wu-ftpd.org/

  • Proftpd

    • 特点：Proftpd 是一个完全独立开发的 FTP 服务器，旨在提供一个安全且易于配置的替代品。它支持多种认证方式和访问控制机制。
    • 官网：http://www.proftpd.org/

  • vsftpd

    • 特点：vsftpd 以其高性能和稳定性著称，尤其适合需要处理大量并发连接的服务器。它的配置文件简单，易于管理。
    • 官网：http://vsftpd.beasts.org/

FTP 服务的使用者

FTP 支持以下三种认证模式：

  1. 匿名用户模式：允许任何人无需密码直接登录 FTP 服务器。此模式安全性最低，通常只用于存放不重要的公开文件，生产环境中不推荐使用。
  2. 本地用户模式：依赖于 Linux 系统的本地账号进行验证，相较于匿名模式更为安全。
  3. 虚拟用户模式：FTP 服务器专用的用户，限制访问 Linux 系统的其他资源，进一步提升 FTP 服务器的安全性。

FTP 的典型消息

消息号 含义
125 数据连接打开，传输开始
200 命令 OK，表示上一个命令执行成功
226 数据传输完毕，表示文件传输已经成功完成
331 用户名 OK，需要输入密码
425 不能打开数据连接，可能是服务器端口被占用或防火墙阻止了连接
426 数据连接被关闭，传输被中断，可能是网络问题或服务器主动断开连接
452 错误写文件，可能是服务器磁盘满、权限不足或文件被锁定
500 语法错误，不可识别的命令，可能是客户端发送了服务器不支持的命令

vsftpd 的配置文件

vsftpd 的配置文件分为以下几个部分：

  • /etc/vsftpd/vsftpd.conf：vsftpd 的核心配置文件。包含了大部分的配置选项，如匿名用户设置、权限控制、传输模式等。
  • /etc/pam.d/vsftpd ：这是 vsftpd 的 PAM 配置文件，用于配置用户认证方式。PAM（Pluggable Authentication Modules）允许 vsftpd 使用不同的认证模块，如本地用户认证、LDAP 认证等。
  • /etc/vsftpd.ftpusers 或 /etc/vsftpd/ftpusers：黑名单文件，列出的用户将无法访问 FTP 服务器。默认情况下，这些用户包括 root 和其他系统用户，以防止敏感账号被用于 FTP 登录。
  • /etc/vsftpd.user_list 或 /etc/vsftpd/user_list：白名单文件，列出的用户可以访问 FTP 服务器。该文件用于指定允许或禁止访问 FTP 服务器的用户列表。当 userlist_enable 选项启用时，可以通过 userlist_deny 选项控制是允许还是禁止列出的用户访问服务器。

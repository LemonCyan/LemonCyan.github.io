C++ 核心特性与类和对象技术要点整合

C++ 是在 C 语言的基础上开发出来的强类型、面向对象编程语言。本指南详细总结了从 C 语言向 C++ 过渡的关键语法差异、新增特性（命名空间、内联、重载、引用、转型），以及面向对象编程（类与对象、构造/析构函数、初始化列表、this 指针）的底层技术细节。

1. 命名空间与类型系统基本变化

1.1 命名空间 (Namespace) 的设计与使用

为了防范大型工程中成千上万个变量或函数发生的命名冲突，C++ 引入了命名空间机制，用以划分不同的作用域。

1) 命名空间的声明

  • 注意：命名空间的定义末尾不需要加分号（区别于结构体和类）。

2) 三种导入/使用空间成员的方式

  1. **名字空间指令 (Using Directive)**：
    using namespace icbc; — 全透明导入空间所有成员，使用最方便但最容易引起命名冲突。
  2. **名字空间声明 (Using Declaration)**：
    using icbc::save; — 仅导入特定的变量或函数，后续可直接使用 save(100)。
  3. 完全作用域限定访问：
    icbc::draw(50) — 直接使用双冒号 :: 显式限定，最安全。

3) 匿名（无名）命名空间与别名

  • 匿名命名空间：如果一个全局变量或函数没有包含在任何名字空间内，它默认被存放在匿名命名空间中。访问外部全局变量时，使用**无名限定符 ::**（如 ::g_data）可以避免被同名的局部变量所屏蔽。
  • 命名空间别名：对于超长的嵌套命名空间，可起简短的别名以简化调用：

1.2 结构体、联合、枚举的 C++ 升级

1) 结构体 (Struct) 增强

  • 关键字可省：定义结构体变量时，可以直接省略 struct 关键字。例如直接写 Person p;，而非 struct Person p;。
  • 内置函数支持：C++ 结构体中可以直接定义成员函数（与类行为类似），其内部可以直接访问数据成员而不需要传入指针。

2) 联合体 (Union) 增强

  • 匿名联合支持：C++ 支持无名称的联合体定义。匿名联合体中的变量会被直接引入当前作用域中，无需通过联合变量名来间接引用：

3) 枚举 (Enum) 的强类型安全限制

  • 在 C 语言中，枚举本质上是整型常数，可以在它们之间隐式转换（如 color c = 0;）。
  • C++ 强制类型匹配：C++ 中枚举是一种完全独立的数据类型。绝对不允许将普通整型数值直接赋给枚举变量。

1.3 bool 类型与运算符别名

  • bool 类型：占用 1 字节内存。其值仅为 true (1) 或 false (0)。
  • boolalpha 控制符：在输出流中使用 cout << boolalpha << val; 可以将数值 1 / 0 的输出格式自动置换为易读的字符串 "true" / "false"。
  • 运算符别名：C++ 支持对关系/位运算符书写对应的英文单词缩写别名，如用 and 代替 &&，用 or 代替 ||，用 not 代替 ! 等（底层通过预处理头文件 <ciso646> 或编译器内部保留字支持）。

2. 函数重载、缺省参数、哑元与内联

2.1 函数重载 (Overloading) 底层机制

同一作用域中，函数名字相同，但形参列表（类型、个数、顺序）不同的函数构成函数重载。

1) 换名机制 (Name Mangling) 的物理本质

C++ 实现函数重载的物理核心是编译器换名。在编译时，编译器会偷偷把函数名进行改写，将参数列表的信息编码进新的函数签名中。

  • 例如，void show(int) 可能会被改名为 _Z4showi，而 void show(double) 被改名为 _Z4showd。因此在底层的机器码中，它们仍然是两个完全独立的函数。
  • extern “C” 的救赎：如果希望 C++ 编译器在编译某些函数时保持原汁原味的 C 风格（不进行任何改名，以供 C 语言程序调用），应使用 extern "C" 限制声明：

2) 重载函数的匹配优先级关系

当调用重载函数时，编译器会根据实参类型进行最优选择匹配，其优先级从高到低为：

  1. **完全匹配 (Exact Match)**：实参与形参类型完全一致，包括通过 const 引用匹配。
  2. **提升转换 (Promotion)**：例如 char、short 提升为 int，float 提升为 double。
  3. **标准转换 (Standard Conversion)**：例如 int 转为 double，或者指针转为 void*。
  4. **自定义转换 (User-Defined Conversion)**：使用类中重载的转换构造函数。
  5. **省略号转换 (…)**：可接收任意参数，匹配度最低。

2.2 缺省参数 (Default Arguments) 约束

C++ 允许在声明函数时，给参数设定默认缺省值。

  1. 从右向左缺省原则：如果一个参数有缺省值，那么它右侧的所有参数都必须有缺省值。
  2. 声明与定义的分离：如果函数的声明和定义分开书写，缺省值必须写在声明部分，定义部分绝不能再写缺省值。
  3. 重载歧义防范：当使用缺省参数时，必须避免与无参或少参的重载函数产生调用歧义：

2.3 哑元函数与内联函数 (Inline)

1) 哑元 (Dummy Variable)

哑元是指只有数据类型，没有变量名称的形参。例如 void fun(int)。

  • 主要用途：
    1. 用于向下兼容早期历史代码中的接口参数变更。
    2. 用于区分重载运算符中的自增自减（如 operator++() 代表前自增，而 operator++(int) 带一个 int 哑元代表后自增）。

2) 内联函数 (Inline Functions)

使用 inline 关键字修饰的函数。

  • 优势：编译器会将该函数的二进制指令直接在调用点展开，消除了常规函数调用中压栈、保护现场和返回的开销。
  • 与预处理宏 #define 的区别：宏定义只是简单的文本替换，容易因为优先级产生副作用；而内联函数是真正的函数，支持严格的数据类型检查与编译调试。
  • 非强制性约束：inline 只是对编译器的“请求”，如果函数体过大或包含递归/循环，编译器会自动忽略内联请求，将其退化为普通函数调用。

4. 引用机制硬性规则与类型转换操作符

4.1 C++ 引用的硬性约束与传递本质

1) 引用定义的六大硬性约束

引用 (Reference) 是既存变量的别名，并不是独立的对象，物理上共享同一块内存。

  1. 定义时必须立刻初始化，不能留空。
  2. 一旦初始化完成，无法解除绑定或重新指向。任何对引用的赋值都会直接改写其绑定对象的值。
  3. 非常量引用（Type &）的初始值必须是可修改的左值变量，不能绑定到常量、临时变量或字面值上。
  4. 引用的类型必须与其绑定的对象类型严格一致。
  5. 不能定义引用的引用（传统 C++ 没有二级引用）。
  6. 多变量定义时的独立性：每一行定义多个引用时，每个引用名前面都必须独立加上 & 修饰符（如 int &r1 = i, &r2 = j;）。

2) 类型匹配的两个特例

  • **常量引用 (const Type &)**：是“万能引用”，允许绑定到字面值、临时表达式甚至不同类型但可隐式转换的对象。编译器会在后台创建一个临时变量进行中转。
  • 继承体系：基类的引用可以直接绑定到派生类的对象上（实现多态的基础）。

4.2 C++ 四大显式类型转换操作符 (Casts)

为了替代 C 语言中粗暴且不安全的强转括号 (TYPE)val，C++ 引入了四个针对性极强的类型转换操作符：

1) 静态类型转换 static_cast

  • 语法：目标类型 变量 = static_cast<目标类型>(源变量);
  • 使用范围：用于在编译器能进行隐式类型转换的类型之间进行转换（如 int 转 double，void * 转具体的 int * ）。如果两个方向都无法隐式转换，则编译器会直接报错，比 C 强转更安全。

2) 常量类型转换 const_cast

  • 语法：目标类型 变量 = const_cast<目标类型>(源变量);
  • 使用范围：唯一能去除或增加变量指针/引用的 const 或 volatile 属性的强转符。

3) 重解析类型转换 reinterpret_cast

  • 语法：目标类型 变量 = reinterpret_cast<目标类型>(源变量);
  • 使用范围：在指针与整型之间，或者完全不相关的任意指针类型之间执行底层的逐字节重解释转换。此操作不执行任何类型检查，属于高危操作。

4) 动态类型转换 dynamic_cast

  • 语法：目标类型 变量 = dynamic_cast<目标类型>(源变量);
  • 使用范围：用于含有虚函数的多态基类与子类指针/引用之间的安全向下转型。它会在运行时进行 RTTI 检查，转换失败时指针返回 nullptr，引用抛出异常。

5. 类与对象、构造/析构函数与生命周期管理

5.1 类与对象的基本概念与区别

  • **类 (Class)**：是对现实世界中对象的属性（数据成员）和行为（成员函数）的抽象提取与模板定义。
  • **对象 (Object)**：是类在内存中分配的物理实体实例。
  • 类与结构体的唯一区别：
    • 结构体 (struct) 的默认访问控制级别是 **public**。
    • 类 (class) 的默认访问控制级别是 **private**。

5.2 构造函数与初始化列表

1) 构造函数 (Constructor) 约束

  • 构造函数的名称必须与类名完全相同。
  • 绝对没有返回值，甚至不能写 void。
  • 如果类中没有自定义任何构造函数，编译器会自动生成一个无参的缺省构造函数；一旦显式定义了任何带参构造函数，编译器就不再提供默认无参构造函数。

2) 初始化列表的物理机制与执行顺序

初始化列表的语法是：类名(形参1, 形参2): 成员1(形参1), 成员2(形参2) { }。

[!IMPORTANT]
初始化列表的两个核心规范：

  1. 执行优先级高于构造函数体：成员变量的真实初始化是在进入构造函数 {} 之前完成的。
  2. 初始化顺序与列表中的书写顺序完全无关，而是严格取决于成员变量在类体内部的定义/声明顺序！
❌ 典型致命陷阱：
  • 崩溃分析：因为 m_len 在类体内部定义在 m_str 之前，所以编译器会**先初始化 m_len**。此时 m_str 还是一个尚未分配和赋值的空对象，调用 m_str.length() 会直接引发段错误（Segment fault）或获取随机的垃圾长度值！
  • 修复方案：将列表项改为 m_len(str.length()) 或调整类内部成员变量的定义顺序。

3) 必须使用初始化列表的场景

  1. 类成员变量中包含 const 常量成员。
  2. 类成员变量中包含引用（&）类型成员。
  3. 派生类（子类）的构造函数需要显式初始化父类部分的成员。

5.4 析构函数 (Destructor) 与资源管理

1) 析构函数规范

  • 名称必须是：**~类名()**。
  • 无参数，无返回值，无法进行重载，一个类只能有一个析构函数。
  • 生命周期：在对象生命周期结束（如局部变量离开作用域，或堆对象被 delete）时，系统会自动触发析构函数的调用。

2) 深度析构与动态资源释放

默认的缺省析构函数在退出时不会主动释放那些通过指针动态申请的系统外部资源（如堆内存、打开的文件描述符、建立的 socket 套接字连接等）。为了防范内存泄漏与死锁，只要类内部占用了动态资源，就必须显式编写析构函数并在其内部执行释放操作。

6. C++ std::string 标准字符串与 char_traits 字符特征

在 C++ 中，std::string (本质是 std::basic_string<char>) 是标准库提供的字符串类，相比于 C 风格的 char[]，它能够自动管理内存，并提供了极其丰富的操作接口。

6.1 char_traits 字符特征类与包装

std::basic_string 的行为是由字符特征类 char_traits 控制的。它定义了具体字符元素的底层物理操作：

  • **assign(a, b)**：将字符 b 赋值给 a。
  • eq(a, b) 和 **lt(a, b)**：定义字符相等 (==) 和小于 (<) 关系。
  • **compare(p1, p2, n)**：比较两组字符串前 n 个字符的大小（类似于 memcmp）。
  • **length(p)**：计算字符串的有效长度（类似于 strlen）。
  • **copy(p1, p2, n)**：复制字符序列（类似于 memcpy）。
  • **move(p1, p2, n)**：复制可能会重叠的字符序列（类似于 memmove）。

2) 区间边界安全性规则

  • std::string::operator[] 访问不抛出异常：类似于普通数组，如果使用下标 s[i] 越界访问超出有效长度的字符，它不会抛出 out_of_range 异常，而是产生未定义行为（通常读取到随机数据或引发崩溃）。
  • std::string::at(i) 强制边界检查：如果使用 s.at(i) 越界访问，系统会强制抛出 std::out_of_range 异常，可用于安全的代码中。

6.4 内存与缓冲区容量管理 (Capacity)

std::string 内部维护了一块动态内存缓冲区，其管理机制与 std::vector 非常相似：

  • **size() / length()**：返回当前存储的字符实际个数。
  • **capacity()**：返回内部缓冲区在不重新分配内存的情况下可以容纳的最大字符数。
  • **reserve(n)**：为主体显式预留指定大小的缓冲区，防范多次追加导致频繁的内存重分配与拷贝。
  • **resize(n, val)**：重设有效字符个数。如果 n 大于当前大小，多余部分用字符 val 填充；如果小于当前大小，超出部分会被截断丢弃。
  • **max_size()**：返回当前操作系统下 std::string 理论上能分配的最大单体长度上限。

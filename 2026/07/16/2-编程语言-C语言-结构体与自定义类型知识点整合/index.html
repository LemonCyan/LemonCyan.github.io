C 语言结构体与自定义数据类型核心知识点整合

结构体 (Structure) 是 C 语言中极其重要的自定义复合数据类型。它允许程序员将不同类型的数据项组合成一个有机的整体，是实现模块化封装和模拟面向对象开发的重要基石。

2.1 指向结构体变量的指针

定义一个指向结构体对象的指针，其指针的类型是该结构体类型本身。

1) 声明与内存分配

指针变量 p1 自身仅占用 4 字节（32位下）或 8 字节（64位下）的地址内存。如果未对其进行初始化赋值或通过 malloc 申请物理存储空间，它就是一个野指针，任何对其成员的解引用写操作都会触发程序段错误崩溃。

2.2 结构体成员引用操作符：. 与 ->

  • 成员运算符 . (Dot)：用于结构体类型变量本身。
    • 语法：结构体变量名.成员名
  • 指向运算符 -> (Arrow)：用于结构体指针变量。
    • 语法：结构体指针名->成员名
    • 指向运算符 p1->num 在编译器底层物理等价于：(*p1).num。

3. 结构体类型数组与指针引用

定义一个结构体类型数组时，数组名 student 代表整个数组的首地址。我们可以使用结构体指针指向并遍历该数组。

对于结构体数组中第 i 个元素的成员的访问，以下三种方法完全等价：

3.1 1) 地址法（行指针偏移）

利用指针或数组名进行地址偏移计算，然后通过指向符访问：

  • (student + i)->name
  • (p + i)->name
  • (&student[i])->name

3.2 2) 指针步进法（自增移位）

3.3 3) 指针的数组下标表示法

当指针 p 指向数组 student 首地址后，我们可以直接把 p 当作数组名，利用变址运算符 [] 进行下标访问。

[!WARNING]
下标运算 p[i] 代表的是第 i 个结构体变量本身，因此访问数组成员时必须使用 . (点运算符)，而绝对不能使用 ->。

  • **p[i].name**（等价于 student[i].name）

5. 面向对象编程 (OOP) 在 C 语言中的实现

面向对象编程是一种设计思想，而不是某种语言的专属特权。利用 C 语言的结构体、函数指针和聚合组合等特性，我们同样可以优雅地实现面向对象的三大基本特征：封装、继承和多态。

5.1 封装 (Encapsulation) 的 C 语言模拟

封装的本质是将“数据成员”与“成员函数”绑定在一起，限制外部对内部数据的随意改动。

  • 在 C++ 中，类（Class）自带 this 指针，成员函数隐式获得对象地址。
  • 在 C 中，我们通过在结构体中定义成员变量 + 函数指针来模拟类，并通过在调用成员函数时显式传递对象地址（作为第一个参数）来模拟 this 指针。

示例：封装一个 Person 类

5.2 继承与多态 (Inheritance & Polymorphism) 的 VTable 模拟

  • 继承：通过在派生类（子类）结构体中嵌套基类（父类）实例的引用或结构体来实现。
  • 多态（虚函数）：通过基类结构体中声明一个 void *pDerivedObj 属性指向子类对象，并在子类构造函数中改写（Override）基类的函数指针位置指向子类成员函数（即模拟 C++ 的虚函数表 VTable 覆盖）。

示例：Employee 继承自 Person 并覆盖 Display 虚函数

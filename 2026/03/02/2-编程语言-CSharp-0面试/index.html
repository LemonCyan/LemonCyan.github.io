1、事务
​ 事务执行一系列的数据库语句内容，要么一起成功要么一起回滚（撤销成功的语句）
​ 四大特性：
​ ①原子性：一起执行，
​ ②一致性：数据库状态执行前后一致
​ ③持久性：一旦成功永久保存在数据库中
​ ④隔离性：并发事务彼此互不干扰

2、单例模式
​是一种创建型设计模式，旨在确保一个类只有一个实例，并提供全局访问点以获取该实例。这在需要限制类的实例数量，确保全局状态一致性，或避免重复资源消耗的情况下非常有用。（类在全局只有一个实例，节省资源）
特点：单一实例： 单例模式确保一个类只能有一个实例。该实例由类自己管理，并在需要时进行创建。
全局访问点： 单例模式提供一个全局访问方法，允许其他代码在任何时候获取该唯一实例。
四种常用的创建方式：
经典的懒汉式（Lazy Initialization）： 在首次使用时创建实例。需要注意的是，多线程环境下可能引发并发问题，需要进行同步处理。
线程安全的懒汉式（Thread-Safe Lazy Initialization）： 在多线程环境下保证线程安全，但可能会带来性能开销。
饿汉式（Eager Initialization）： 在类加载时就创建实例，不存在多线程问题，但可能在程序启动时带来性能开销。
双重检查锁定（Double-Check Locking）： 在多线程环境下保证线程安全，同时尽量减少锁的使用，提高性能。

4、索引
​索引（Index）是一种数据结构，用于加速数据库表中数据的检索操作。通过创建索引，数据库可以更快地定位和访问表中的特定数据行，而不必进行全表扫描。索引通常基于表中的一个或多个列，提高了查询效率，但同时也会占用一定的存储空间并对写操作（插入、更新、删除）产生一定的影响。
（1）索引类型：
​ ①唯一索引（Unique Index）： 唯一索引要求索引列中的每个值都是唯一的，用于确保数据的唯一性。例如，一个表中的主键通常会自动创建唯一索引。
​ ②聚集索引（Clustered Index）： 聚集索引决定了数据在磁盘上的物理存储顺序。每个表只能有一个聚集索引，通常是主键索引。表的数据按照聚集索引的顺序存储，因此聚集索引对于范围查询效率较高。
​ ③非聚集索引（Non-clustered Index）： 非聚集索引不会改变数据在磁盘上的物理存储顺序，而是创建一个单独的索引结构来加速数据的检索。一个表可以有多个非聚集索引。
（2）SQL SERVER 索引实例：
①创建一个普通非聚集索引
CREATE NONCLUSTERED INDEX idx_FirstName ON Customers (FirstName);
②创建一个唯一非聚集索引：
CREATE UNIQUE NONCLUSTERED INDEX idx_CustomerID_unique ON Customers (CustomerID);
③创建一个聚集索引（一般用于主键）：
CREATE CLUSTERED INDEX idx_CustomerID ON Customers (CustomerID);
（3）MYSQL 索引实例：

  1. C#中值类型和引用类型有什么区别？请举例说明。

答：值类型存储在栈上，包括基本数据类型（如 int, double, bool）、结构体（struct）和枚举（enum），赋值时进行值拷贝；引用类型存储在堆上，包括类（class）、接口（interface）、数组（array）和字符串（string），赋值时传递引用。int a = 10; int b = a; // b 是 a 的副本，修改 a 不会影响 b。而对于类，如 StringBuilder sb1 = new StringBuilder(“test”); StringBuilder sb2 = sb1; sb1.Append(“123”); // sb2 的值也会变为”test123”，因为两者指向同一对象。

  1. C#中 string 和 StringBuilder 的区别？

答：string 是不可变的，每次对 string 进行修改（如拼接、替换）都会创建一个新的 string 对象，原有对象会被标记为垃圾回收，适合少量字符串操作场景；StringBuilder 是可变的，内部维护一个可动态扩容的字符数组，修改操作直接在该数组上进行，不会频繁创建新对象，高效适用于大量字符串拼接、修改等操作场景。

  1. C#中 ref 和 out 关键字有什么区别？

答：从参数初始化要求看，ref 关键字要求传递的参数在传递给方法前必须完成初始化；out 关键字则不要求参数在传递前初始化，但强制要求在方法内部必须为 out 参数赋值。从使用场景看，ref 用于需要将已初始化的变量传入方法，方法可能会读取并修改该变量的值后反馈给调用方；out 主要用于方法需要返回多个值的场景，通过 out 参数承载额外的返回结果。

  1. C#中 is 和 as 运算符的区别？

答：功能定位不同，is 运算符用于检查某个对象是否与指定类型兼容（包括该类型及其派生类型），执行后返回 bool 类型结果（true 或 false）；as 运算符用于将对象转换为指定的引用类型或可空值类型，转换成功则返回对应的对象，转换失败时不会抛出异常，而是返回 null（仅适用于引用类型和可空值类型，不能用于值类型转换）。

  1. C#中如何实现可空值类型？

答：有两种实现方式，一种是使用泛型结构体 Nullable<T>（T 为值类型），另一种是使用简化的?修饰符。例如，要表示可空的整数类型，可写成 Nullable<int> 或 int?，两种写法等效，可空值类型允许变量除了存储对应值类型的正常取值外，还能存储 null 值。

面向对象

  1. C#中抽象类（abstract class）和接口（interface）的区别？

答：首先，成员实现要求不同，抽象类可以包含抽象成员（无实现）和非抽象成员（有具体实现）；C# 8.0 之前接口只能包含抽象成员，C# 8.0 及以后支持默认实现，但接口的核心定位仍是规范。其次，继承规则不同，抽象类遵循单继承原则，一个类只能继承一个抽象类；接口支持多实现，一个类可以同时实现多个接口。最后，设计意图不同，抽象类用于表示“is-a”的继承关系，体现类之间的共性和继承层次；接口用于表示“can-do”的能力契约，强调类具备的特定功能。

  1. C#中虚方法（virtual）和抽象方法（abstract）的区别？

答：实现要求不同，虚方法必须有默认的实现代码，定义时需搭配方法体；抽象方法没有实现代码，定义时只需声明方法签名，且必须用 abstract 关键字修饰。使用场景不同，虚方法用于父类提供基础实现，同时允许子类根据需求通过 override 关键字重写该方法（子类可重写也可不重写）；抽象方法用于父类定义方法规范，强制要求非抽象子类必须通过 override 关键字重写该方法实现。另外，包含抽象方法的类必须声明为抽象类，而包含虚方法的类可以是普通类。

  1. C#中 sealed 关键字的作用？

答：sealed 关键字有两种主要作用，一是用于修饰类，此时该类被称为密封类，表示该类不能被其他类继承，防止类的继承层次被扩展；二是用于修饰类中的虚方法（virtual 方法），此时该方法不能被子类重写，即使子类继承了包含该密封方法的类，也无法修改该方法的实现。需要注意的是，sealed 修饰方法时，该方法必须是重写（override）的方法，不能直接修饰普通方法。

  1. C#中如何实现多态？

答：C#中多态主要通过两种方式实现，分别对应编译时多态和运行时多态。编译时多态（静态多态）通过方法重载（overload）实现，指在同一个类中或父子类中，定义多个方法名相同但参数列表（参数个数、类型、顺序）不同的方法，编译器在编译时根据调用方法时传入的参数信息确定要执行的具体方法。运行时多态（动态多态）通过虚方法（virtual）和方法重写（override）实现，父类定义虚方法并提供基础实现，子类通过 override 关键字重写该方法，当使用父类引用指向子类对象并调用该虚方法时，程序会在运行时根据对象的实际类型执行对应的子类重写方法。

  1. C#中构造函数和析构函数的执行顺序？

答：构造函数的执行顺序是“先父后子”，当创建子类对象时，会先调用父类的构造函数（默认先调用父类的无参构造函数，若父类无无参构造函数则需在子类构造函数中显式指定调用父类的有参构造函数），完成父类成员的初始化后，再调用子类的构造函数，初始化子类自身的成员。析构函数的执行顺序与构造函数相反，是“先子后父”，当对象被垃圾回收时，会先执行子类的析构函数，释放子类相关的资源，之后再执行父类的析构函数，释放父类相关的资源。析构函数由垃圾回收器（GC）自动调用，无需手动调用。

代理异步

  1. C#中委托（delegate）和事件（event）的区别？

答：本质关系上，事件是基于委托实现的，事件内部封装了委托，是委托的一种特殊应用场景。访问权限控制不同，委托是一种类型安全的函数指针，在声明它的类外部可以直接调用委托对象来触发关联的方法；事件采用发布-订阅模式，对外暴露的操作被限制，在声明事件的类外部只能通过+=（订阅）和-=（取消订阅）操作来关联或解除方法，无法直接调用事件触发方法，只能由声明事件的类内部通过特定逻辑触发，安全性更高。设计意图不同，委托主要用于封装方法，实现方法的传递和回调；事件主要用于实现对象间的通信，让订阅者响应发布者的特定状态变化。

  1. C#中 Action 和 Func 委托的区别？

答：核心区别在于是否有返回值。Action 委托用于封装无返回值的方法，它是一个泛型委托，支持 0 到 16 个参数，可以根据方法的参数列表选择对应的 Action 泛型版本，例如 Action（无参数）、Action<int>（一个 int 类型参数）、Action<string, int>（两个参数，分别为 string 和 int 类型）等。Func 委托用于封装有返回值的方法，同样是泛型委托，支持 0 到 16 个参数，且最后一个类型参数固定表示方法的返回值类型，例如 Func<int>（无参数，返回 int 类型）、Func<string, int>（一个 string 类型参数，返回 int 类型）等。

  1. C#中 async 和 await 关键字的作用？

答：async 和 await 关键字搭配使用，用于简化异步方法的编写和调用。async 关键字用于标记一个方法为异步方法，告诉编译器该方法内部可能包含 await 表达式，并且编译器会将该方法转换为一个状态机来管理异步操作的执行流程，异步方法的返回值通常为 Task、Task<TResult>（有返回值）或 void（主要用于事件处理程序，不推荐常规异步方法使用）。await 关键字用于等待一个异步操作（返回 Task 或 Task<TResult> 的操作）完成，使用 await 时不会阻塞当前线程，而是会将方法的执行权交还给调用方，当异步操作完成后，程序会从暂停的位置继续执行后续代码，使异步代码的编写风格接近同步代码，更易读易懂。

  1. C#中 Task 和 Thread 的区别？

答：抽象层次不同，Thread 是操作系统级别的线程，直接对应操作系统的内核线程，开发者需要直接管理线程的创建、启动、销毁等操作，控制粒度较细。Task 是基于.NET 线程池的更高层次的抽象，属于托管线程的封装，不需要开发者直接与操作系统内核线程交互，由.NET 框架负责线程的调度和管理。资源开销不同，创建和销毁 Thread 的开销较大，因为涉及到操作系统内核资源的分配和释放，频繁创建销毁 Thread 会影响程序性能。Task 利用线程池中的线程进行复用，减少了线程创建销毁的开销，能更高效地管理线程资源，适合大量并发任务的场景。另外，Task 支持更丰富的功能，如任务的取消、延续（ContinueWith）、等待多个任务完成（Task.WhenAll、Task.WhenAny）等，而 Thread 的功能相对单一。

  1. C#中如何取消一个异步操作？

答：主要通过 CancellationTokenSource（CTS）和 CancellationToken（CT）组合实现。首先创建一个 CancellationTokenSource 对象，该对象可以生成 CancellationToken；然后将生成的 CancellationToken 传递给需要取消的异步方法（异步方法需支持接收 CancellationToken 参数，并在内部定期检查该令牌的 IsCancellationRequested 属性）；当需要取消异步操作时，调用 CancellationTokenSource 的 Cancel()方法，此时对应的 CancellationToken 的 IsCancellationRequested 属性会变为 true；异步方法内部通过检查该属性，若发现已请求取消，则可以通过抛出 OperationCanceledException 异常或直接退出方法等方式来终止异步操作。例如：var cts = new CancellationTokenSource(); var task = DoAsyncWork(cts.Token); // 取消操作 cts.Cancel();

多线程

  1. C#中 lock 关键字的作用？

答：lock 关键字用于实现多线程环境下的线程同步，确保同一时刻只有一个线程能够进入被 lock 包裹的代码块（临界区）执行。其核心作用是防止多个线程同时访问共享资源（如共享变量、对象等）而导致的数据竞争和数据不一致问题。lock 关键字的使用语法为 lock（锁定对象）{ 临界区代码 }，锁定对象通常建议使用 private static readonly 修饰的对象，避免锁定字符串常量或值类型（会导致装箱操作，可能引发锁定失效问题）。lock 本质上是 Monitor 类的语法糖，等效于 Monitor.Enter（锁定对象）和 Monitor.Exit（锁定对象）的组合，并且会自动处理异常，确保即使临界区代码抛出异常，也能释放锁资源。

  1. C#中 Monitor 和 lock 的区别？

答：关系上，lock 是 Monitor 的语法糖，lock（obj）代码块在编译后会被转换为 Monitor.Enter(obj)、try 代码块（包含临界区代码）和 finally 代码块（包含 Monitor.Exit(obj)）的组合，确保锁的正确释放。功能丰富度不同，Monitor 提供了比 lock 更丰富的功能，例如 Monitor.TryEnter 方法可以尝试获取锁，并指定超时时间，若在超时时间内未获取到锁则返回 false，避免线程无限期等待；而 lock 没有超时机制，线程会一直等待直到获取到锁。另外，Monitor 还提供了 Monitor.Wait、Monitor.Pulse、Monitor.PulseAll 等方法，用于实现线程间的协作（如生产者-消费者模式），而 lock 不支持这些协作功能。使用灵活性不同，lock 使用简单直观，适合常规的线程同步场景；Monitor 使用相对复杂，但能满足更复杂的同步需求。

  1. C#中线程池（ThreadPool）的作用？

答：线程池的核心作用是管理一组可复用的线程资源，避免频繁创建和销毁线程带来的高额性能开销。线程池在程序启动时会预先创建一定数量的线程（最小线程数），当有任务需要执行时，线程池会从池中分配一个空闲线程来执行任务，任务执行完成后，该线程不会被销毁，而是返回线程池成为空闲线程，等待后续任务的分配。当任务数量超过池中空闲线程数量时，线程池会根据设置的最大线程数限制创建新的线程来处理任务；当任务数量减少，池中空闲线程数量超过最小线程数时，线程池会销毁部分空闲线程，以节省系统资源。线程池还能自动管理线程的调度，提高线程的使用效率，适合处理大量短期、轻量级的并发任务，如异步 I/O 操作后的回调任务、网络请求处理等。

  1. C#中如何实现线程同步？

答：C#提供了多种线程同步原语来实现线程同步，常见的有以下几种：一是 lock 关键字/Monitor 类，lock 是 Monitor 的语法糖，用于确保同一时刻只有一个线程进入临界区，适合简单的同步场景；二是 Mutex（互斥体），属于跨进程的同步原语，不仅能实现同一进程内多个线程的同步，还能实现不同进程间的线程同步，使用时需要手动调用 WaitOne 方法获取锁和 ReleaseMutex 方法释放锁；三是 Semaphore（信号量），用于控制同时访问临界区的线程数量，通过设置信号量的初始计数和最大计数来实现，调用 WaitOne 方法计数减 1（获取权限），调用 Release 方法计数加 1（释放权限）；四是 EventWaitHandle（事件等待句柄），包括 AutoResetEvent 和 ManualResetEvent，用于实现线程间的通知机制，通过 Set 方法发送信号、WaitOne 方法等待信号来实现同步；五是 Interlocked 类，提供了原子操作方法（如 Increment、Decrement、Add 等），用于对共享变量进行原子性的读写操作，避免数据竞争。

  1. C#中 Volatile 关键字的作用？

答：Volatile 关键字主要用于修饰字段，其作用是确保该字段的读写操作不会被编译器或 CPU 进行优化重排，并且每次对该字段的读取操作都能从内存中获取最新的值，而不是从 CPU 缓存（如一级缓存、二级缓存）中获取缓存的旧值。在多线程环境下，若多个线程同时访问同一个未被 Volatile 修饰的字段，由于编译器优化或 CPU 缓存的存在，可能会导致线程读取到该字段的旧值，出现数据不一致的问题。例如，一个线程修改了字段的值，该值可能先存储在 CPU 缓存中，尚未同步到内存，而另一个线程读取该字段时从内存中获取到的仍是修改前的值。通过给字段添加 Volatile 修饰符，可以避免这种问题，保证字段读写的可见性和有序性。需要注意的是，Volatile 不能保证复合操作（如 i++）的原子性，仅适用于简单的读写操作。

IO 操作

  1. C#中如何使用 FileStream 读写文件？

答：使用 FileStream 读写文件通常遵循以下步骤：首先，创建 FileStream 对象，创建时需要指定文件路径（path）、文件打开方式（FileMode，如 Create、Open、Append 等）、文件访问权限（FileAccess，如 Read、Write、ReadWrite），还可根据需求指定文件共享方式（FileShare）等参数；然后，进行读写操作，若读取文件，可创建 byte 数组作为缓冲区，调用 FileStream 的 Read 方法将文件内容读取到缓冲区中；若写入文件，同样准备好要写入的 byte 数组，调用 FileStream 的 Write 方法将缓冲区内容写入文件；读写完成后，必须调用 FileStream 的 Close 方法关闭流，释放文件资源，为了确保资源可靠释放，推荐使用 using 语句包裹 FileStream 对象，using 语句会在代码块执行完毕后自动调用 Dispose 方法关闭流。using (FileStream fs = new FileStream(“test.txt”, FileMode.Create, FileAccess.Write)) { byte[] data = Encoding.UTF8.GetBytes(“Hello World”); fs.Write(data, 0, data.Length); }

  1. C#中 StreamReader 和 StreamWriter 的作用？

答：StreamReader 和 StreamWriter 是.NET 提供的文本读写工具类，它们封装了字节流（如 FileStream），专门用于处理文本文件的读写操作，无需开发者手动进行字节与字符串之间的编码转换。StreamReader 的作用是读取文本文件，它可以从指定的字节流中读取字符，并支持指定编码格式（如 UTF8、GB2312 等），提供了 ReadLine（读取一行文本）、ReadToEnd（读取全部文本）等便捷方法，简化了文本读取流程，避免了直接操作字节流的繁琐。StreamWriter 的作用是写入文本文件，它可以将字符串转换为指定编码的字节后写入到字节流中，同样支持指定编码格式，提供了 Write（写入字符串）、WriteLine（写入一行字符串并换行）等方法，方便进行文本内容的写入，并且也推荐使用 using 语句包裹，确保资源释放。例如，使用 StreamReader 读取文本：using (StreamReader sr = new StreamReader(“test.txt”, Encoding.UTF8)) { string content = sr.ReadToEnd(); }

  1. C#中如何异步读取文件？

答：异步读取文件可通过两种常见方式实现，均需结合 async 和 await 关键字。第一种方式是使用 FileStream 的异步方法，创建 FileStream 对象后（需确保 FileStream 支持异步操作，创建时可指定 useAsync 参数为 true），调用 ReadAsync 方法异步读取文件内容，该方法返回 Task<int>，表示读取到的字节数，通过 await 等待读取完成。第二种方式是使用 StreamReader 的异步方法，创建 StreamReader 对象后，调用其 ReadLineAsync（异步读取一行）或 ReadToEndAsync（异步读取全部内容）等异步方法，这些方法返回 Task<string>，通过 await 等待获取读取结果。两种方式都需将调用方法标记为 async，并且推荐使用 using 语句管理流对象，确保资源释放。public async Task ReadFileAsync() { using (StreamReader sr = new StreamReader(“test.txt”, Encoding.UTF8)) { string content = await sr.ReadToEndAsync(); Console.WriteLine(content); } }

  1. C#中如何使用 BinaryReader 和 BinaryWriter？

答：BinaryReader 和 BinaryWriter 是用于读写二进制文件的工具类，它们封装了字节流，提供了直接读写基本数据类型（如 int、double、bool、string 等）的方法，无需开发者手动处理数据的二进制编码。使用 BinaryReader 读取二进制文件的步骤：创建 BinaryReader 对象，传入对应的字节流（如 FileStream）和编码格式；调用其 ReadInt32、ReadDouble、ReadString 等方法读取对应类型的数据；读取完成后关闭 BinaryReader，推荐使用 using 语句。使用 BinaryWriter 写入二进制文件的步骤：创建 BinaryWriter 对象，传入对应的字节流和编码格式；调用其 Write 方法（支持多种基本数据类型重载）写入数据；写入完成后关闭 BinaryWriter，同样推荐 using 语句。// 写入二进制文件 using (BinaryWriter bw = new BinaryWriter(new FileStream(“data.bin”, FileMode.Create), Encoding.UTF8)) { bw.Write(100); bw.Write(3.14); bw.Write(“test”); } // 读取二进制文件 using (BinaryReader br = new BinaryReader(new FileStream(“data.bin”, FileMode.Open), Encoding.UTF8)) { int intVal = br.ReadInt32(); double doubleVal = br.ReadDouble(); string strVal = br.ReadString(); }

  1. C#中如何监控文件系统的变化？

答：通过 FileSystemWatcher 类实现文件系统变化的监控。使用步骤如下：首先，创建 FileSystemWatcher 对象，指定要监控的目录路径（Path 属性），若要监控子目录需设置 IncludeSubdirectories 属性为 true；然后，设置要监控的变化类型，通过 Filter 属性指定监控的文件类型（如”.txt”监控文本文件，””监控所有文件），通过 NotifyFilter 属性指定要监控的通知类型（如 FileName、DirectoryName、LastWrite 等，即文件名变化、目录名变化、最后写入时间变化等）；接着，为 FileSystemWatcher 的相关事件注册事件处理程序，常见事件有 Created（文件/目录创建）、Changed（文件/目录修改）、Deleted（文件/目录删除）、Renamed（文件/目录重命名）；最后，设置 EnableRaisingEvents 属性为 true，启动监控。需要注意的是，FileSystemWatcher 依赖于操作系统的文件系统通知机制，可能存在部分场景下通知丢失的情况，且监控时需确保程序有对应目录的访问权限。FileSystemWatcher watcher = new FileSystemWatcher(@”C:\Test”); watcher.IncludeSubdirectories = true; watcher.Filter = “.txt”; watcher.NotifyFilter = NotifyFilters.FileName | NotifyFilters.LastWrite; watcher.Created += (sender, e) => Console.WriteLine($”文件创建：{e.FullPath}”); watcher.EnableRaisingEvents = true;

其他

  1. C#中 using 关键字的作用？

答：C#中 using 关键字有两种主要作用。第一种作用是引入命名空间，通过 using 命名空间;的语法，使该命名空间下的类、接口等成员可以直接使用类名访问，无需写完整的命名空间前缀，简化代码编写，例如 using System;后可直接使用 Console 类，无需写 System.Console。第二种作用是实现资源的自动释放，用于包裹实现了 IDisposable 接口的对象，语法为 using（创建实现 IDisposable 接口的对象）{ 对象的使用代码 }，当 using 代码块执行完毕后，无论代码块内是否抛出异常，都会自动调用该对象的 Dispose 方法，释放对象占用的非托管资源（如文件句柄、数据库连接、网络连接等），避免资源泄漏。例如，使用 using 包裹 FileStream 对象：using (FileStream fs = new FileStream(“test.txt”, FileMode.Open)) { // 操作文件流 }

  1. C#中 IDisposable 接口的作用？

答：IDisposable 接口的核心作用是提供一种统一的机制，用于释放对象所占用的非托管资源。托管资源（如类的实例、数组等）由.NET 的垃圾回收器（GC）自动管理和回收，但 GC 无法自动识别和释放非托管资源（如文件句柄、数据库连接、COM 对象、网络套接字等）。当一个类持有非托管资源时，就需要实现 IDisposable 接口，该接口只包含一个 Dispose 方法，开发者在 Dispose 方法中编写释放非托管资源的代码。使用该类时，通过调用 Dispose 方法（或结合 using 关键字自动调用），即可主动释放非托管资源，避免资源长时间占用导致的资源泄漏问题。实现 IDisposable 接口时，通常还会搭配析构函数（终结器），作为 Dispose 方法未被调用时的兜底方案，确保非托管资源最终能被释放。

  1. C#中 GC（垃圾回收）的工作原理？

答：GC（垃圾回收器）是.NET 框架提供的自动内存管理机制，主要负责回收堆内存中不再被使用的对象，释放内存资源，避免内存泄漏。其核心工作原理基于“标记-清除-压缩”算法，并采用分代回收策略提高效率。首先是标记阶段，GC 会从应用程序的根对象（如静态变量、当前线程栈上的局部变量、CPU 寄存器中的引用等）出发，遍历所有可达的对象，将这些可达对象标记为“存活”；然后是清除阶段，GC 扫描整个堆内存，将未被标记为“存活”的对象（即垃圾对象）进行回收，释放其占用的内存空间；最后是压缩阶段（针对大对象堆以外的堆区域），GC 将存活的对象向堆的一端移动，消除内存碎片，使后续内存分配更高效。分代回收策略将对象分为 0 代、1 代、2 代，新创建的对象为 0 代，回收频率最高；经过一次回收未被回收的对象升级为 1 代，回收频率较低；经过 1 代回收未被回收的对象升级为 2 代，回收频率最低，通过这种方式提高垃圾回收的效率。

  1. C#中如何避免内存泄漏？

答：避免内存泄漏可从以下几个方面入手：一是及时释放非托管资源，对于持有非托管资源的类，务必实现 IDisposable 接口，在 Dispose 方法中释放非托管资源，并在使用该类时结合 using 关键字确保 Dispose 方法被调用，或手动调用 Dispose 方法；二是避免静态集合长期持有对象引用，静态集合的生命周期与应用程序一致，若将对象添加到静态集合后未及时移除，即使对象不再被其他地方使用，也会因被静态集合引用而无法被 GC 回收，需在对象不再需要时从静态集合中移除；三是合理使用事件，订阅事件后若不再需要响应事件，需及时取消订阅（使用-=操作符），否则发布者会一直持有订阅者的引用，导致订阅者无法被回收；四是使用弱引用（WeakReference），对于一些需要临时引用但又不希望影响对象回收的场景，可使用 WeakReference 来引用对象，GC 在回收对象时不会因弱引用的存在而阻止回收；五是避免无限递归或长时间运行的方法导致栈溢出或对象无法释放，同时注意调试时及时发现内存泄漏问题，可使用 Visual Studio 的内存分析工具等辅助排查。

  1. C#中如何调试多线程程序？

答：调试多线程程序可采用以下方法和工具：一是利用 Visual Studio 的并行调试工具，该工具提供了并行堆栈窗口（可查看所有线程的调用堆栈，直观了解各线程的执行位置）、并行任务窗口（可查看所有 Task 的状态、ID 等信息，方便跟踪任务执行情况）、线程窗口（可查看所有线程的详细信息，如线程 ID、状态、所属进程等，还可手动冻结或解冻线程，控制线程的执行）；二是设置条件断点和命中次数断点，在多线程场景下，普通断点可能会被多个线程触发，通过设置条件断点（如只有当某个变量等于特定值时触发断点）或命中次数断点（如第 N 次命中时触发），可精准定位到目标线程的执行场景；三是使用日志记录，在关键代码位置（如线程启动、进入临界区、修改共享资源等）记录日志，包含线程 ID、时间戳、操作内容等信息，通过分析日志可追溯各线程的执行顺序和操作过程，排查多线程竞争等问题；四是利用线程本地存储（ThreadLocal<T>）隔离线程间的变量，减少线程间的干扰，便于定位单个线程的问题；五是重现问题时尽量控制线程数量和执行顺序，简化场景，逐步排查问题根源。

面试题 50

  1. 什么是通用中间语言（CIL）
    CIL（Common Intermediate Language，通用中间语言）是 C#、VB.NET 等.NET 语言编译后的中间代码（也叫 IL），符合 CLI 规范，由 CLR 执行。所有.NET 语言最终都编译成相同的 CIL。

  2. 什么是公共语言运行时（CLR）
    CLR（Common Language Runtime）是.NET 的运行时环境，负责加载程序集、执行 CIL 代码、进行即时编译（JIT）、垃圾回收（GC）、异常处理、线程管理、安全检查等。

  3. C# 和 .NET 的区别是什么
    C# 是一种编程语言（语法规则，由微软设计）；.NET 是运行平台/框架（包括 CLR、基类库 FCL、工具链等）。C#代码编译成 IL 后在.NET 运行时执行。目前主流是统一的“.NET”（以前分 Framework 和 Core）。

  4. 值类型和引用类型有什么区别
    值类型：直接存储数据（栈上或内嵌），赋值时拷贝全部内容，不能为 null（除非 Nullable），无继承（除接口）。
    引用类型：存储引用（地址，堆上对象），赋值时拷贝地址，共享同一对象，可为 null，支持继承、多态。

  5. 什么是装箱和拆箱
    装箱（boxing）：值类型 → object（或接口），在堆上分配新对象并拷贝值（隐式/显式）。
    拆箱（unboxing）：object → 值类型，必须显式转换，类型不匹配抛 InvalidCastException。常见于 ArrayList、非泛型集合。

  6. 三种主要的错误类型是什么
    语法错误（compile-time）、逻辑错误（运行但结果错）、运行时异常（Runtime exceptions，如除零、null 引用）。

  7. C# 中如何处理异常
    使用 try { } catch(Exception ex) { } finally { } 块。
    可捕获特定异常（如 catch(IOException)），finally 始终执行（清理资源）。推荐：using 语句自动 Dispose；全局异常处理（AppDomain/ASP.NET 中间件）。

  8. C# 中有哪些访问修饰符
    public（任意可见）、protected（本类+派生类）、internal（同一程序集）、private（仅本类）、protected internal（同一程序集或派生类）、private protected（同一程序集且派生类，C#7.2+）。

  9. C# 中的默认访问修饰符是什么
    类/记录/结构体：internal
    类成员（字段/方法等）：private
    接口成员：public（C#8 前默认 public）
    枚举/委托：internal

  10. sealed 修饰符的作用是什么
    sealed 类：禁止被继承。
    sealed 方法：禁止被 override（常用于重写后的最终实现）。
    用途：防止进一步派生、保护关键逻辑、性能微优化。

  11. params 关键字的作用是什么
    params int[] args 允许方法接收可变数量的同类型参数（作为数组）。只能是最后一个参数。例：void Print(params string[] names)。

  12. 类和结构体的区别是什么
    class（引用类型，堆分配，可 null，支持继承、多态）；
    struct（值类型，栈/内嵌分配，不可 null，无继承，默认值零初始化，适合小数据结构如 Point、DateTime）。

  13. 什么是分部类
    partial class / partial struct / partial interface / partial void 方法。允许把一个类定义拆分到多个文件（常用于自动生成代码 + 手工代码分离，如 WinForms 设计器）。

  14. new 关键字的作用是什么

    1. 创建对象：new MyClass()
    2. 隐藏基类成员：new void Method()（方法隐藏，非多态）
    3. 泛型约束：where T : new()（要求有无参构造函数）

  15. static 关键字的作用是什么
    static 成员属于类型而非实例：静态字段/方法/属性/构造函数/类。
    共享内存、无需实例、可在 using static 中直接调用。

  16. 什么是静态类
    static class：只能包含静态成员，不能实例化（new 无效），常用于工具类（如 Math、Console）。

  17. 三元条件运算符的作用是什么
    condition ? trueExpr : falseExpr
    简洁的 if-else 替代，常用于赋值或返回。

  18. null 合并运算符和 null 条件运算符的作用是什么
    ??（null 合并）：left ?? right（left 不为 null 则 left，否则 right）
    ?. / ?[]（null 条件/Elvis）：obj?.Property / arr?[index]（obj 为 null 则整条链返回 null，不抛 NullRef）

  19. 什么是封装
    封装（Encapsulation）：隐藏内部实现细节，只暴露必要接口（通过属性、方法）。通过 private/protected 字段 + public 属性实现。

  20. 什么是 LINQ
    Language Integrated Query：集成到 C#的查询语法（查询式/方法式）。支持对内存集合、数据库、XML 等统一查询。例：from x in list where x > 10 select x;

  21. 什么是扩展方法
    static class 中的 static 方法，第一个参数用 this 修饰，可“扩展”已有类型（无需改源代码）。例：public static int WordCount(this string str) { … }

  22. 什么是 IEnumerable
    IEnumerable：表示可枚举的集合，支持 foreach。实现 GetEnumerator()返回 IEnumerator（MoveNext、Current、Reset）。

  23. 等值运算符（==）和 Equals 方法有什么区别
    ==：默认比较引用（引用类型）或值（值类型）。string 重载为值比较。
    Equals()：虚方法，默认同==，但可重写为值比较（常与 GetHashCode 一起重写）。

  24. 深拷贝和浅拷贝的区别是什么
    浅拷贝：只拷贝引用，对象字段仍指向同一对象（MemberwiseClone 默认）。
    深拷贝：递归拷贝所有层级对象，得到完全独立副本（需实现 ICloneable 或序列化）。

  25. 什么是垃圾回收器（Garbage Collector）
    GC 自动管理堆内存：标记-清除-整理（分代：0/1/2 代），回收无引用对象。触发时机：内存压力、显式 GC.Collect（慎用）。

  26. 什么是可空类型
    Nullable 或 T?（值类型可为 null）。例：int? x = null;
    HasValue、Value 属性；?? / GetValueOrDefault()。

  27. 什么是属性（Property）
    封装字段的 get/set 访问器。自动属性：public int Age { get; set; }
    本质：编译成 get_XXX() / set_XXX() 方法。

  28. 什么是泛型
    泛型（Generics）：类型参数化，运行时类型安全，避免装箱/拆箱。例：List、Dictionary<TKey, TValue>。支持约束：where T : class / struct / new() / 接口。

  29. const 和 readonly 修饰符的区别是什么
    const：编译期常量，必须声明时初始化，只能基本类型/string。
    readonly：运行时只读，可在构造函数/初始化器赋值，可为任何类型。

  30. ref 和 out 关键字的区别是什么
    ref：传入/传出（必须初始化）。
    out：只传出（方法内必须赋值，调用前可不初始化）。

  31. 接口和抽象类的区别是什么
    接口：多继承、纯契约（C#8 前无实现）、成员默认 public。
    抽象类：单继承、可有实现字段/方法/属性、成员可非 public。

  32. 什么是多态性
    多态（Polymorphism）：同一接口不同实现。运行时根据实际类型调用 override 方法（虚方法/抽象方法/接口）。

  33. 虚方法和抽象方法的区别是什么
    virtual：有默认实现，可被 override。
    abstract：无实现，必须在非抽象派生类中 override。

  34. 什么是方法重载
    同名方法，不同参数列表（个数/类型/顺序），与返回值无关。编译期决议。

  35. 方法重写和方法隐藏的区别是什么
    override：多态，重写 virtual/abstract 方法。
    new：隐藏基类同签名方法，非多态（静态绑定）。

  36. C# 是否支持多重继承
    不支持类多重继承（只支持单继承），但支持多接口实现。

  37. 什么是 DRY 原则
    Don’t Repeat Yourself：不要重复代码。提取公共逻辑到方法/类/基类/扩展方法。

  38. 什么是“魔法数字”反模式
    代码中直接写硬编码的数字/字符串（如 if(x==42)），缺乏可读性。应定义常量/枚举/配置文件。

  39. 为什么使用 goto 关键字被认为是不良实践
    goto 导致代码难以理解、维护（意大利面条式代码）。现代语言有结构化控制流（break/continue/return）替代。

  40. 什么是意大利面条式代码（Spaghetti Code）
    逻辑混乱、跳转复杂、难以追踪的代码（常因滥用 goto、全局变量、深层嵌套引起）。

41–45. 什么是单例 / 建造者 / 适配器 / 桥接 / 工厂方法 设计模式
单例（Singleton）：确保类只有一个实例，提供全局访问点。
建造者（Builder）：分离复杂对象的构建与表示（一步步构造）。

  1. override 和 new 的区别？（方法隐藏 vs 方法重写）
  • override：多态重写（polymorphism），用于重写基类的 virtual 或 abstract 方法。运行时根据实际对象类型调用（动态绑定）。
  • new：方法隐藏（hiding），隐藏基类同签名方法。编译时根据变量的声明类型决定调用哪个（静态绑定），非多态。
  1. virtual 和 abstract 的区别？
  • virtual：有默认实现，可被子类 override。
  • abstract：无实现（或只有声明），必须在非抽象子类中 override，抽象类/方法不能实例化。
    virtual 适合提供默认行为，abstract 强制子类实现。

  1. C# 中的委托是什么？委托和方法有什么关系？
    委托（Delegate）是类型安全的函数指针，本质是一个引用方法的类（可指向一个或多个方法）。
    关系：委托实例可以“指向”符合签名的方法，调用委托就像调用方法。支持多播（+=/-=）。
    delegate void Action(string s); Action del = Console.WriteLine; del("hi");

  2. 事件和委托的区别？事件为什么用 event 关键字？

  • 委托：公开的、可直接赋值/调用/+=的函数指针。
  • 事件：对委托的封装，只能在声明类内部触发（Invoke），外部只能订阅（+=）/取消（-=），防止外部误调用或清空。
    event 关键字提供封装保护：外部看不到委托的完整功能（如=null、Invoke）。
  1. 泛型是什么？泛型约束有哪几种？
    泛型（Generics）：用类型参数（T/U 等）编写可复用、类型安全的代码，避免装箱/类型转换。
    常见约束（where）：
  • class / struct：引用/值类型
  • new()：必须有无参公共构造函数
  • 接口/基类：必须实现/继承某类型
  • notnull / unmanaged（C# 8+）
  • default（C# 11+）
  1. using 语句的作用是什么？（除了引入命名空间还有什么）
  • 引入命名空间：using System;
  • 资源管理：using (var obj = new Disposable()) { } 自动在作用域结束时调用 Dispose()（即使异常）。

  1. IDisposable 接口有什么用？using 和 Dispose 的关系？
    IDisposable：提供 Dispose() 方法，用于确定性释放非托管资源（文件句柄、数据库连接等）和昂贵托管资源。
    using 语句：语法糖，等价于 try-finally，在 finally 中调用 Dispose()（如果实现了 IDisposable）。

  2. GC（垃圾回收）是怎么工作的？分代回收是什么？
    GC：自动回收堆上无引用的对象。
    过程：标记（从根开始找可达对象）→ 清除（回收不可达）→ 压缩（整理碎片，Gen2 可能）。
    分代（Generation）：0（新生代，频繁 GC，快）、1（中间）、2（老年代，慢但少）。新对象在 Gen0，存活多次晋升代际，减少全堆扫描。

  3. 强引用、弱引用、软引用、虚引用分别是什么？

  • 强引用（默认）：GC 只有无强引用才回收。
  • 弱引用（WeakReference）：允许 GC 回收对象，即使还有弱引用（用于缓存）。
  • 软引用（SoftReference，.NET 无原生，Java 有）：内存不足时优先回收。
  • 虚引用（PhantomReference，.NET 无原生）：仅用于跟踪 finalize 前通知，几乎不用。
  1. == 和 Equals() 的区别？什么时候需要重写？
  • **==**：运算符，默认比较引用（引用类型）或值（值类型）。string 重载为值比较。
  • **Equals()**：虚方法，默认同==，但可重写为值比较。
    需要重写：自定义类/结构体用作 Dictionary 键、HashSet 元素、或需值相等比较时，必须同时重写 Equals + GetHashCode。

  1. GetHashCode() 和 Equals() 为什么要一起重写？
    Dictionary/HashSet 等哈希集合依赖：如果两个对象 Equals 相等，则 GetHashCode 必须返回相同值。
    不一起重写 → 哈希碰撞或找不到对象（违反相等契约）。

  2. StringBuilder 和 string 的性能区别？什么时候用哪个？

  • string：不可变，每次+或 Concat 创建新对象（O(n²)风险）。
  • StringBuilder：可变缓冲区，Append 高效（O(1)均摊）。
    用 StringBuilder：大量字符串拼接（循环中>5 次以上）。
    用 string：少量拼接、常量、简单场景（编译器优化）。
  1. ArrayList 和 List 的区别？
  • ArrayList：非泛型，存 object → 装箱/拆箱、类型不安全、性能差。
  • List：泛型，类型安全、无装箱、性能更好。
    现代几乎只用 List。

  1. Dictionary 的底层实现？为什么键要重写 GetHashCode 和 Equals？
    底层：哈希表（数组 + 链表/红黑树）。
    键调用 GetHashCode()定位桶 → Equals()解决冲突。
    不重写 → 默认引用比较 → 自定义对象键无法正常查找/存储。

  2. foreach 内部是怎么实现的？（IEnumerable 和 IEnumerator）
    foreach 要求类型实现 IEnumerable（或 IEnumerable）。
    内部：调用 GetEnumerator()得到 IEnumerator → while(MoveNext()) → 使用 Current。
    yield return 自动生成状态机实现 IEnumerator。

  • 方法语法：Where().OrderBy().Select()

  1. var 和 dynamic 的区别？
  • var：编译期类型推断，静态类型（强类型），性能同显式类型。
  • dynamic：运行时绑定，绕过编译检查（类似弱类型），可能运行时异常，性能差（反射）。
    var 用于本地变量简化，dynamic 用于 COM、动态 JSON 等。

  1. async 和 await 的底层实现原理？（状态机）
    async 方法编译成状态机结构体（IAsyncStateMachine）。
    await 处拆分成状态：0=初始、1=挂起、2=完成。
    遇到 await → 返回未完成 Task，状态机保存上下文 → 延续时恢复状态继续执行。

  2. Task 和 Thread 的区别？

  • Thread：底层 OS 线程，创建/销毁昂贵，手动管理。
  • Task：高层抽象（基于线程池），支持 async/await、取消、异常传播、组合（WhenAll/WhenAny）、返回值。
    现代优先用 Task（TPL）。

  3. 值类型和引用类型在方法参数传递时的区别？（ref、out、in）

  2. 问题：C#中的值类型和引用类型有什么区别？string 是哪种类型？
    值类型和引用类型的主要
    值类型（如 int, struct）直接存储数据，分配在栈上；
    引用类型（如 class, string）存储对数据的引用，分配在堆上；
    值类型赋值时会复制整个对象，引用类型赋值时只复制引用；
    值类型不会产生垃圾回收压力，引用类型会产生 GC 压力。
    string 是引用类型，但它具有值类型的语义特征：当修改 string 变量时，会创建新的 string 对象，而不是修改原对象。

    Lambda 表达式是 C# 3.0 中对匿名方法的简化语法，提供了更简洁的函数定义方式。
    语法简洁性：Lambda 表达式语法更简洁；
    类型推断：Lambda 表达式支持更好的类型推断；
    表达式体：Lambda 表达式可以直接使用表达式作为函数体；
    应用场景：Lambda 表达式更适用于 LINQ 查询。

// 匿名方法（C# 2.0）
button.Click += delegate(object sender, EventArgs e) {
MessageBox.Show(“按钮被点击”);
};

// Lambda 表达式（C# 3.0）
button.Click += (sender, e) => MessageBox.Show(“按钮被点击”);

// 用于 LINQ
var evenNumbers = numbers.Where(n => n % 2 == 0);

  1. 问题：C# 7.0 中的 out 变量有什么优势？

out 变量是 C# 7.0 引入的特性，允许在方法调用时直接声明 out 参数变量，使代码更加简洁。

优势：

简化 out 参数的使用；
减少变量声明的冗余代码；
提高代码的可读性；
支持隐式类型推断。

// 旧方式
int result;
if (int.TryParse(“123”, out result))
{
Console.WriteLine($”解析成功: {result}”);
}

// 新方式
if (int.TryParse(“123”, out int result))
{
Console.WriteLine($”解析成功: {result}”);
}

// 使用 var 关键字
if (int.TryParse(“123”, out var result))
{
Console.WriteLine($”解析成功: {result}”);
}

public struct LargeStruct
{

    // 可以读取 other 的值
    int result = this.Value1 + other.Value1;
    
    // 不能修改 other
    // other.Value1 = 10; // 编译错误
}

}

表达式体成员是 C# 6.0 引入的特性，提供了一种更简洁的语法来定义只包含单个表达式的成员。

适用场景：

简单的只读属性；

public class Person
{

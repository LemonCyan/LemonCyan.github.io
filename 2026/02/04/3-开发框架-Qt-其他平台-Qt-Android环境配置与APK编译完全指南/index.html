Qt Android 环境配置与 APK 编译完全指南

环境要求

Qt 6 Android 开发需要以下组件，版本之间存在严格依赖：

组件 Qt 6.5+ 版本要求 说明
JDK 17+ 推荐 Adoptium 或 OpenJDK 发行版，64 位
Android SDK API 28-36 需包含 Platform Tools 和 Build Tools
Android NDK 匹配 Qt 版本的 NDK Qt 6.10 使用 NDK r27c (27.2.12479018)
Gradle 8.x Qt 6.10 集成 Gradle 8.14.3 + AGP 8.10.1

[!IMPORTANT]
NDK 版本必须与 Qt 官方构建时使用的版本匹配，否则会出现符号链接错误。错误的 NDK 版本是最常见的编译失败原因之一。

第一部分：开发环境搭建

1.1 安装 JDK

Qt 6.x 要求 JDK 17 或更高版本。推荐从以下来源获取：

  • Adoptium（推荐）
  • OpenJDK
  • Oracle JDK

1.2 安装 Android SDK 和 NDK

有两种安装方式：通过 Qt Creator 自动安装（推荐）或手动安装。

方式一：Qt Creator 自动配置（推荐）

  1. 打开 Qt Creator → Edit → Preferences → Devices → Android
  2. 点击 Set Up SDK 按钮
  3. Qt Creator 将自动下载并配置：
    • Android Command-line SDK Tools
    • Platform Tools
    • Build Tools
    • 匹配当前 Qt 版本的 NDK

方式二：手动安装

下载地址：

  • SDK Tools: https://developer.android.com/studio#command-line-tools-only
  • NDK: https://developer.android.com/ndk/downloads

Linux 自动化安装脚本：

Windows PowerShell 安装脚本：

1.3 配置 Qt Creator

完成 SDK/NDK 安装后，在 Qt Creator 中配置路径：

  1. Tools → Options → Devices → Android

  2. 设置以下路径：

    • JDK 位置：C:\Program Files\Eclipse Adoptium\jdk-17.0.x 或 /usr/lib/jvm/java-17-openjdk-amd64
    • Android SDK 位置：SDK 安装目录
    • Android NDK 位置：NDK 版本目录（需指向具体版本，如 ndk/27.2.12479018）

  3. Tools → Options → Kits

    • 添加或选择 Android Qt 版本
    • 配置对应的 Kit，选择正确的编译器和 Qt 版本

第五部分：常见问题排查

5.1 ABI 检测失败

错误： Failed to detect the ABIs used by the Qt version

原因： Qt Android 组件未正确安装

解决：

  1. 运行 Qt Maintenance Tool → Add or Remove Components
  2. 确保选中 Qt for Android 相关组件
  3. 重启 Qt Creator

5.2 NDK 目录结构错误

错误： NDK directory structure isn't correct

原因： NDK 路径指向了 ndk 根目录而非版本目录

解决：
将路径从 /path/to/ndk 改为 /path/to/ndk/27.2.12479018（具体版本号）

5.3 Gradle 构建失败

错误： Execution failed for task ':processDebugResources'

解决：

  1. 检查 android-build/build.gradle 中的 AGP 版本
  2. 确保 Gradle 版本与 AGP 兼容
  3. 清理构建目录后重试：

5.4 Windows 路径过长

错误： FileNotFoundException 或构建失败

原因： Windows 默认路径限制 260 字符

解决：

  • 将项目移动到较短的路径（如 C:\Dev\MyApp）
  • 或启用 Windows 长路径支持

5.5 版本不匹配

问题表现 可能原因 解决方法
链接错误 NDK 版本不匹配 使用 Qt 对应的 NDK 版本
Gradle 错误 AGP 版本过低 更新 build.gradle
编译失败 JDK 版本不正确 Qt 6.x 需要 JDK 17+

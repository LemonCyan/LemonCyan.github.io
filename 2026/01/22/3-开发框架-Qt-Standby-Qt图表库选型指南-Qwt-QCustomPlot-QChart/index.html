Qt图表库选型指南：Qwt、QCustomPlot、Qt Charts 深度对比

Qwt、QCustomPlot、Qt Charts 是 Qt 生态中最常用的三个图表库。它们的架构差异直接影响性能上限和集成成本，许可证策略也各不相同。本文汇总官方文档、GitHub Issues 和社区实测数据，给出选型所需的关键信息。

快速选型

场景 推荐 理由
科学仪器 / 工业控制 Qwt LGPL 许可、外部缓冲区、仪器控件库
快速原型 / 美观优先 QCustomPlot 双文件集成、默认样式好看
QML 原生 (Qt 6.8+) Qt Graphs RHI 后端、官方推荐
存量商业项目 Qt Charts 仅限维护已有代码

关键差异：

  • 许可证：Qwt 采用 LGPL 兼容协议，商业闭源更友好；QCustomPlot 和 Qt Charts 是 GPL v3
  • 性能：Qwt > QCustomPlot >> Qt Charts（后者受 Graphics View 架构拖累）
  • 集成：QCustomPlot 仅需 2 个文件，Qwt 需编译库，Qt Charts 需模块配置
  • 实时刷新：Qwt 和 QCustomPlot 稳定支持 100Hz+，Qt Charts 超过 10Hz 就会卡

[!IMPORTANT]
Qt Charts 在 Qt 6.10 正式废弃，官方建议新项目用 Qt Graphs。

差异对比

维度 Qwt 6.3.x QCustomPlot 2.1.x Qt Charts
许可证 LGPL 兼容 (Qwt License 1.0) GPL v3 (商业需购买) GPL v3 / 商业
集成方式 编译静态/动态库 双文件 (.h + .cpp) Qt 模块
渲染后端 QPainter (可选 OpenGL) QPainter (可选 OpenGL) Graphics View
适用场景 仪器、工业控制 实时监控、原型 商业报表
Qt 版本 4.8 / 5.x / 6.x 4.7+ / 5.x / 6.4 5.x / 6.x (6.10 废弃)
QML 支持 需嵌入 QWidget 需嵌入 QWidget 原生组件

性能瓶颈在这里。QGraphicsScene 的事件分发开销大，每个数据点对应一个 QGraphicsItem，内存占用高，也没有硬件加速的自适应采样。

选型决策树

工业监控

推荐 Qwt。

典型场景：8-32 通道波形、1kHz+ 采样率、频谱图/瀑布图、7×24 运行。

Qwt 的 QwtPlotSpectrogram 原生支持频谱和热力图，内存效率高。QwtWheel、QwtKnob 等工业控件也是现成的。示波器、频谱仪用它的很多。

快速原型

推荐 QCustomPlot。

典型场景：周期性报表、10-100Hz 曲线监控、赶工期、样式要好看。

两个文件丢进项目就能用，不用配构建系统。默认样式比较现代，交互功能也全。社区活跃，示例多。

QML 应用

推荐 Qt Graphs（Qt 6.8+）。

适合静态或低频更新图表、需要 QML 深度集成、饼图柱状图之类的商业图表。

[!WARNING]
Qt Charts 已废弃。老代码里如果用了它，注意 GPL v3 许可证对商业项目的限制。

局限性

Qwt

问题 描述 解法
Windows 编译麻烦 没预编译包，要手动配 qmake 看 qwt.pro 配置
QML 嵌入差 基于 QWidget 用 QQuickWidget 包装
上手慢 API 偏底层 从 examples 目录学起
3D 弱 QwtPlot3D 停更了 换 Qt Data Visualization

QCustomPlot

问题 描述 解法
长跑内存涨 数据点累积 做滑动窗口
QML 里慢 嵌进去帧率掉 单独开 QWidget 窗口
GPL 限制 商业闭源要买许可 联系作者
百万点撑不住 超 100 万点就慢 降采样

Qt Charts

问题 描述 解法
废弃了 6.10 起不维护 迁到 Qt Graphs
架构限制 Graphics View 开销大 没救
几千点就卡 大数据集撑不住 换 QCustomPlot
实时差 高频刷新吃不消 批量 replace + 降帧率

怎么选

  • 许可证敏感 → Qwt（LGPL）
  • 性能优先 → Qwt ≥ QCustomPlot >> Qt Charts
  • 快速集成 → QCustomPlot（两文件）> Qt Charts > Qwt
  • QML 原生 → Qt Graphs >> 其他

结合决策树和实际约束权衡即可。

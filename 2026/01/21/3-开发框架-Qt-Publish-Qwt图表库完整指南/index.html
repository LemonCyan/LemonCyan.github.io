Qwt 6.x 图表库完整指南

Qwt (Qt Widgets for Technical Applications) 是一个基于 Qt 的 C++ GUI 组件库，主要用于科学和工程领域。除了二维绘图组件，它还提供刻度尺、滑块、表盘等工业控件。当前稳定版本 Qwt 6.3.0，支持 Qt 5.6+ 和 Qt 6.x。

一、核心架构

1.1 类层次结构

Qwt 绘图系统以 QwtPlot 为核心，使用组合模式管理图表元素。

组件指责：

组件 职责 生命周期
QwtPlot 容器，负责坐标轴与图元的布局 随父窗口销毁
QwtPlotCanvas 渲染层，处理绘制和输入 由 QwtPlot 管理
QwtPlotItem 图元（曲线、标记、网格） attach() 后归 QwtPlot 所有
QwtScaleEngine 刻度计算引擎 通过 setAxisScaleEngine() 设置

1.2 渲染流程

replot() 是更新图表的入口。它会先重算坐标轴刻度，标记缓存失效，然后在 paintEvent 中遍历图元进行绘制。

四、问题排查

4.1 编译链接

错误 原因 解决
undefined reference to 'QwtPlot::...' 没链接上 查 .pro 的 LIBS，确认 x86/x64 匹配
qwt_compat.h: No such file 引用了旧头文件 Qwt 6.2+ 删了这个文件，改代码吧
LNK2019 (MSVC) Debug/Release 混用 Debug 用 qwtd.lib，Release 用 qwt.lib

4.2 运行时

现象 检查
曲线不显示 1. attach() 了吗？
2. replot() 了吗？
3. 数据在坐标轴范围内吗？
对数轴崩溃 数据里有 $\le 0$ 的数（对数不能处理）
Designer 无插件 qwt_designer_plugin.dll 没放对位置

4.3 Qt 兼容性

6.1.x ✓ ✗ 旧项目
6.2.0 ✓ ⚠ 部分支持 过渡
6.3.0 ✓ ✓ 推荐

五、选型：Qwt 还是 Qt Charts?

5.1 上手对比

特性 Qwt Qt Charts
协议 LGPL (闭源友好) GPL / 商业许可
场景 科学绘图、工业控制 商业报表、金融K线
性能 优秀 (支持 OpenGL) 良好
交互 内置 Zoomer/Panner 需要自己写
曲线 自带插值算法 基础线条
集成 需编译 Qt 自带

5.2 如何选择

  1. 用 Qwt：如果你做医疗仪器、示波器、工控软件，或者必须用 LGPL 协议。
  2. 用 Qt Charts：如果你只需要画个漂亮的柱状图或饼图，并且接受 GPL 或买了 Qt 商业版。

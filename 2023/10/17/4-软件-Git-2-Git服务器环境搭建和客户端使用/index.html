2. Git服务器环境搭建和客户端使用

服务端

  1. 安装 git 和 ssh
  1. 增加 git 用户并生成文件夹
  1. 创建 ssh 证书认证文件

临时修改文件的权限 sudo chmod 777 /home/git/.ssh/authorized_keys 把需要访问 git 服务器的客户端公钥 id_rsa.pub的内容复制到 authorized_keys 文件

  1. 修改 authorized_keys 文件的权限

  1. 为了安全考虑禁止登录 git 服务器的 shell，修改 git 的 shell 用 /usr/bin/git-shell 把 /etc/passwd 的 git:x:1004:1004:,,,:/home/git:/bin/bash 改成： git:x:1004:1004:,,,:/home/git:/usr/bin/git-shell 保存

  2. 建代码仓库

以后每创建一个新的仓库，记得最后一步操作: 修改仓库所属用户为 git。

客户端

  1. 安装 git
  • Linux 环境下 sudo apt-get install git
  • Windows 环境下直接安装 Git安装包
  1. 配置连接
  • 通过密钥方式 ssh-keygen -t rsa [-C "的邮箱地址"] 会生成 id_rsa.pub 文件,添加该公钥到到服务器
    • Linux 环境下，密钥默认位于 /home/ubuntu/.ssh/id\_rsa
    • Windows 环境下密钥位于 C:\Users\xxx.ssh\id\_rsa.pub
  • 通过用户名/密码

在连接 git 时，会需要输入账号密码，直接输入即可

附注：增量备份 -Git 服务器备份

通过 rsync 使用 crontab 建立每天凌晨 3 点定时触发的任务 crontab -e
